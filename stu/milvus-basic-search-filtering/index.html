Milvus基础搜索与过滤查询详解

Mar 02, 2026
阅读时长: 7 分钟
English

覆盖内容：基础 ANN 搜索（单向量、批量）、过滤搜索（标准过滤 & 迭代过滤）、分页搜索 官方文档：Basic ANN Search | Filtered Search

一、核心概念：ANN 搜索

Milvus 使用 ANN（Approximate Nearest Neighbor） 近似最近邻搜索，而非精确的 kNN 搜索。

对比项kNNANN
原理遍历全部向量基于索引文件定位子集
精度100% 精确近似，可调
性能慢，资源消耗大快，适合大规模数据
适用场景小数据集生产环境

搜索流程：

  1. 根据索引文件快速定位候选子集
  2. 用指定的距离度量（metric_type）计算相似度
  3. 按相似度排序，返回 top-K 结果

距离度量类型

metric_type相似度趋势值域适用场景
COSINE值越大越相似[-1, 1]文本语义搜索（最常用）
IP（内积）值越大越相似[-1, 1]归一化向量
L2（欧氏距离）值越小越相似[0, ∞)图像特征
JACCARD值越小越相似[0, 1]二值向量
HAMMING值越小越相似[0, dim]二值向量

⚠️ 关键点：distance 字段的含义取决于 metric_type。COSINE/IP 下 distance 越大越好；L2 下 distance 越小越好。

二、前置准备：集合加载

搜索前必须确保集合已加载到内存，否则会报错。

def ensure_collection_loaded():
    """确保集合已加载到内存，搜索前必须调用"""
    load_state = client.get_load_state(collection_name=collection_name)

    # state 是枚举类型（LoadState），str(LoadState.Loaded) == 'Loaded'
    if str(load_state.get('state')) != 'Loaded':
        client.load_collection(
            collection_name=collection_name,
            # load_fields 可选：只加载需要的字段，节省内存
            # 不指定则加载全部字段
            load_fields=["book_id", "title", "content_vector", "keyword_vector"]
        )

load_fields 的取舍：

  • 指定 load_fields：节省内存，但 output_fields 只能包含已加载的字段
  • 不指定：加载全部字段，灵活但内存占用大
  • 向量字段（用于搜索的字段）必须包含在 load_fields 中

三、单向量搜索

最基础的搜索方式，传入一个查询向量，返回最相似的 top-K 结果。

def single_vector_search():
    """
    单向量搜索：一次查询，返回最相似的 K 条记录

    search() 返回值结构：
    res[0]          → 第一个查询向量的结果列表
    res[0][i]       → 第 i 个命中结果
    res[0][i]['id'] → 命中实体的主键
    res[0][i]['distance'] → 与查询向量的距离（相似度）
    res[0][i]['entity']   → 命中实体的字段值（由 output_fields 决定）
    """
    # 生成 256 维查询向量（实际场景中来自 embedding 模型）
    query_vector = [float(random.uniform(-1, 1)) for _ in range(256)]

    result = client.search(
        collection_name=collection_name,
        data=[query_vector],           # 单向量时传一维列表，多向量时传二维列表
        anns_field="content_vector",   # 指定在哪个向量字段上搜索
        limit=3,                       # 返回最相似的前 3 条
        search_params={"metric_type": "COSINE"},
        output_fields=["book_id", "title", "rating"]  # 需要返回的标量字段
    )

    # result 是二维列表：result[查询向量索引][命中结果索引]
    for i, hit in enumerate(result[0]):
        print(f"{i+1}. ID: {hit['entity']['book_id']}, "
              f"标题: {hit['entity']['title']}, "
              f"相似度: {hit['distance']:.4f}")

常见误区：

  • limit 最大值受集合配置限制，默认上限 16384

四、批量向量搜索

一次请求传入多个查询向量，Milvus 并行处理，效率高于多次单独搜索。

def bulk_vector_search():
    """
    批量搜索：一次请求处理多个查询向量

    适用场景：
    - 批量推荐（同时为多个用户推荐）
    - 批量相似度计算
    - 减少网络往返次数
    """
    # 准备 3 个查询向量
    query_vectors = [
        [float(random.uniform(-1, 1)) for _ in range(256)]
        for _ in range(3)
    ]

    result = client.search(
        collection_name=collection_name,
        data=query_vectors,   # 传入多个向量
        anns_field="content_vector",
        limit=2,
        search_params={"metric_type": "COSINE"},
        output_fields=["book_id", "title"]
    )

    # result 是二维列表，result[i] 对应第 i 个查询向量的结果
    print(f"共 {len(result)} 组结果")  # → 3
    for i, hits in enumerate(result):
        print(f"\n查询向量 {i+1} 的结果:")
        for j, hit in enumerate(hits):
            print(f"  {j+1}. {hit['entity']['title']}, 相似度: {hit['distance']:.4f}")

💡 性能提示：批量搜索的总耗时通常远小于多次单独搜索的耗时之和，推荐在需要多次搜索时使用。

五、过滤搜索

在向量搜索的基础上，加入标量字段的过滤条件，缩小搜索范围。

5.1 两种过滤方式对比

标准过滤（Standard Filtering）：
  ┌─────────────┐    ┌──────────────┐    ┌──────────┐
  │  全量数据   │ → │ 标量过滤筛选 │ → │ ANN搜索  │ → top-K
  └─────────────┘    └──────────────┘    └──────────┘
  先过滤，再搜索。过滤条件简单时性能好。

迭代过滤（Iterative Filtering）：
  ┌─────────────┐    ┌──────────────┐    ┌──────────────┐
  │  全量数据   │ → │  ANN 迭代搜索 │ → │ 逐条标量过滤 │ → top-K
  └─────────────┘    └──────────────┘    └──────────────┘
  先搜索，再逐条过滤。过滤条件复杂时性能好。
对比项标准过滤迭代过滤
过滤时机ANN 搜索前ANN 搜索后
适用场景过滤后剩余数据量大过滤后剩余数据量小
启用方式默认，无需额外参数search_params 中加 "hints": "iterative_filter"
潜在问题过滤条件复杂时延迟高满足条件的数据极少时需大量迭代，性能下降

5.2 标准过滤

def filtered_vector_search():
    """标准过滤搜索：先过滤标量字段，再在过滤结果中做 ANN 搜索"""
    query_vector = [float(random.uniform(-1, 1)) for _ in range(256)]

    result = client.search(
        collection_name=collection_name,
        data=[query_vector],
        anns_field="content_vector",
        limit=3,
        search_params={"metric_type": "COSINE"},
        # filter 支持的操作符：==, !=, >, >=, <, <=, and, or, not, like, in, array_contains 等
        filter="is_available == True and rating >= 4.0",
        output_fields=["book_id", "title", "is_available", "rating"]
    )

    for i, hit in enumerate(result[0]):
        entity = hit['entity']
        print(f"{i+1}. {entity['title']}, "
              f"在馆: {entity['is_available']}, "
              f"评分: {entity['rating']}, "
              f"相似度: {hit['distance']:.4f}")

常用过滤表达式：

# 布尔字段
filter="is_available == True"

# 数值范围
filter="rating >= 4.5"
filter="price > 50 and price < 200"

# 字符串模糊匹配（% 为通配符）
filter='title like "%Python%"'

# IN 列表
filter="book_id in [1, 2, 3]"

# 复合条件
filter="is_available == True and rating >= 4.0 and price < 100.0"

# JSON 字段访问
filter='details["author"] == "张三"'

# ARRAY 字段包含
filter='array_contains(tags, "编程")'

5.3 迭代过滤

def iterative_filtered_search():
    """
    迭代过滤搜索：适合复杂过滤条件

    原理：ANN 搜索迭代返回候选结果，每个候选结果逐一做标量过滤，
    直到凑够 limit 条满足条件的结果为止。

    优势：减少标量过滤的工作量（只过滤 ANN 返回的候选，不过滤全量数据）
    劣势：如果满足条件的数据极少，需要大量迭代，性能反而下降
    """
    query_vector = [float(random.uniform(-1, 1)) for _ in range(256)]

    result = client.search(
        collection_name=collection_name,
        data=[query_vector],
        anns_field="content_vector",
        limit=3,
        search_params={
            "metric_type": "COSINE",
            "hints": "iterative_filter"   # ← 唯一区别，启用迭代过滤
        },
        # 复杂过滤条件（多字段、多条件）更适合迭代过滤
        filter="is_available == True and rating >= 4.0 and price < 100.0",
        output_fields=["book_id", "title", "is_available", "rating", "price"]
    )

    for i, hit in enumerate(result[0]):
        entity = hit['entity']
        print(f"{i+1}. {entity['title']}, "
              f"评分: {entity['rating']}, "
              f"价格: ¥{entity.get('price', 'N/A')}, "
              f"相似度: {hit['distance']:.4f}")

5.4 如何选择过滤方式

两种方式的核心差异在于过滤后剩余数据的比例：

  • 过滤后剩余数据量大（大多数数据能通过过滤）→ 标准过滤
  • 过滤后剩余数据量小（只有少量数据能通过过滤）→ 迭代过滤

💡 实践建议：不确定时，可以像 filter-search-demo.py 中的 performance_comparison() 那样，实际测量两种方式的耗时，根据数据分布选择。

六、分页搜索

当需要获取超过 limit 上限的结果，或实现翻页功能时，使用 offset 参数。

def pagination_search():
    """
    分页搜索：通过 offset 跳过前 N 条结果，实现翻页

    limit + offset 的总和不能超过 16384（Milvus 默认限制）
    """
    query_vector = [float(random.uniform(-1, 1)) for _ in range(256)]

    page_size = 2  # 每页 2 条

    for page in range(3):  # 获取前 3 页
        offset = page * page_size

        result = client.search(
            collection_name=collection_name,
            data=[query_vector],
            anns_field="content_vector",
            limit=page_size,
            search_params={
                "metric_type": "COSINE",
                "offset": offset    # ← offset 在 search_params 里
            },
            output_fields=["book_id", "title"]
        )

        print(f"\n第 {page+1} 页（offset={offset}）:")
        for i, hit in enumerate(result[0]):
            print(f"  {i+1}. {hit['entity']['title']}")

分页注意事项：

注意点说明
offset 位置可以放在 search_params 字典中，也可以作为 search() 的顶层关键字参数
上限限制limit + offset <= 16384，超出会报错
一致性问题分页期间如有数据写入，不同页的结果可能不一致
性能offset 越大，性能越差（需要跳过前 N 条结果）

⚠️ 大数据量翻页：如果需要遍历大量数据（如导出），推荐使用 Search Iterator（搜索迭代器），性能更好，后续 Part 会介绍。

七、output_fields 使用技巧

# 返回指定字段
output_fields=["book_id", "title", "rating"]

# 返回所有标量字段（不含向量字段）
output_fields=["*"]

# 返回向量字段（数据量大，按需使用）
output_fields=["book_id", "content_vector"]

# 访问 JSON 字段的子属性
output_fields=["details"]

八、搜索结果结构速查

result = client.search(...)

# result 结构
result                    # List[List[Hit]]，外层对应每个查询向量
result[0]                 # 第一个查询向量的所有命中结果
result[0][0]              # 第一个命中结果（最相似）

hit = result[0][0]
hit['id']                 # 实体主键（快捷访问）
hit['distance']           # 与查询向量的距离/相似度
hit['entity']             # 字典，包含 output_fields 指定的字段
hit['entity']['title']    # 访问具体字段值
