一、范围搜索（Range Search）

1.1 核心概念

普通 ANN 搜索返回"最相似的 K 条"，但不限制相似度区间。范围搜索通过 radius 和 range_filter 两个参数，划定一个环形相似度区间，只返回落在区间内的结果。

以查询向量为圆心，画两个同心圆：

  range_filter（内圆）── 排除过于相似的结果（如去重场景）
  radius（外圆）──────── 排除过于不相似的结果

  只返回落在环形区域内的向量

为什么需要内圆（range_filter）？

  • 去重场景：排除与查询向量几乎完全相同的结果（可能就是查询向量本身）
  • 推荐场景：排除用户已经看过的高度相似内容，推荐"相关但不重复"的内容
  • 如果不需要内圆，将 range_filter 设为 0（L2/JACCARD/HAMMING）或不设置即可

1.2 参数设置规则（重点，容易搞混）

不同距离度量类型，参数大小关系相反：

metric_type相似度趋势参数关系含义
COSINE值越大越相似radius < distance <= range_filterradius 是下限，range_filter 是上限
IP值越大越相似radius < distance <= range_filter同上
L2值越小越相似range_filter <= distance < radiusrange_filter 是下限，radius 是上限
JACCARD值越小越相似range_filter <= distance < radius同上
HAMMING值越小越相似range_filter <= distance < radius同上

记忆口诀：

  • COSINE/IP：值大相似 → radius（小）< distance <= range_filter（大）
  • L2/JACCARD/HAMMING：值小相似 → range_filter（小）<= distance < radius（大）

1.3 代码示例

def range_search_demo():
    """
    范围搜索：限定相似度区间，只返回落在区间内的结果

    参数位置：radius 和 range_filter 放在 search_params["params"] 里
    注意：不同 metric_type 下参数大小关系不同，见上方表格
    """
    query_vector = [float(random.uniform(-1, 1)) for _ in range(256)]

    # ── COSINE 距离：radius < distance <= range_filter ──
    # 目标：相似度在 0.8 ~ 0.95 之间的图书（高相似度区间）
    result = client.search(
        collection_name=collection_name,
        data=[query_vector],
        anns_field="content_vector",
        limit=3,
        search_params={
            "metric_type": "COSINE",
            "params": {
                "radius": 0.8,        # 外圆下限（COSINE 下是相似度下限）
                "range_filter": 0.95  # 内圆上限（COSINE 下是相似度上限）
            }
        },
        output_fields=["book_id", "title", "rating"]
    )

    print(f"高相似度搜索结果（相似度 0.8~0.95）: 共 {len(result[0])} 条")
    for i, hit in enumerate(result[0]):
        print(f"  {i+1}. {hit['entity']['title']}, 相似度: {hit['distance']:.4f}")

    # ── 中等相似度区间 ──
    result2 = client.search(
        collection_name=collection_name,
        data=[query_vector],
        anns_field="content_vector",
        limit=3,
        search_params={
            "metric_type": "COSINE",
            "params": {
                "radius": 0.5,
                "range_filter": 0.7   # 相似度 0.5~0.7
            }
        },
        output_fields=["book_id", "title"]
    )
    print(f"中等相似度搜索结果（相似度 0.5~0.7）: 共 {len(result2[0])} 条")

1.4 常用场景参数参考（COSINE）

场景radiusrange_filter说明
去重检测0.951.0找几乎相同的内容
高相似推荐0.80.95强相关内容
中等相似推荐0.50.8相关但有差异
宽松匹配0.30.6泛相关内容

⚠️ 注意：如果参数设置过严（区间太窄），可能返回 0 条结果。建议先用宽松区间测试，再逐步收紧。

二、分组搜索（Grouping Search）

2.1 核心概念

普通 ANN 搜索可能返回大量来自同一"类别"的结果，导致结果同质化。分组搜索按指定字段分组，每组只返回最相似的 N 条，保证结果多样性。

典型场景：RAG 文档检索

问题：一篇长文档被切成 100 个 chunk 存入 Milvus
      普通搜索 top-5 可能全是同一篇文档的不同段落

分组搜索：按 doc_id 分组，每组返回 1 条
          → top-5 来自 5 篇不同文档，结果更多样

2.2 三个关键参数

参数类型说明
group_by_fieldstr按哪个字段分组，支持 BOOL/INT/VARCHAR/ARRAY，不支持 FLOAT
group_sizeint每组最多返回几条，默认 1
strict_group_sizebool是否严格保证每组返回 group_size 条，默认 False

limit 的含义变了：

  • 普通搜索：limit = 返回的实体总数
  • 分组搜索：limit = 返回的分组数，总实体数 ≈ limit × group_size

2.3 代码示例

def grouping_search_demo():
    """
    分组搜索：按字段分组，避免结果同质化

    book_collection 中按 is_available（BOOL）分组：
    - True 组：在馆图书中最相似的
    - False 组：不在馆图书中最相似的
    """
    query_vector = [float(random.uniform(-1, 1)) for _ in range(256)]

    # ── 基础分组：每组返回 1 条 ──
    result = client.search(
        collection_name=collection_name,
        data=[query_vector],
        anns_field="content_vector",
        limit=2,                          # 返回 2 个分组（在馆/不在馆）
        group_by_field="is_available",    # 按是否在馆分组
        group_size=1,                     # 每组返回 1 条
        output_fields=["book_id", "title", "is_available"]
    )
    # 最多返回 2×1=2 条记录

    print(f"基础分组搜索结果: 共 {len(result[0])} 条")
    for hit in result[0]:
        entity = hit['entity']
        status = "在馆" if entity['is_available'] else "不在馆"
        print(f"  [{status}] {entity['title']}, 相似度: {hit['distance']:.4f}")

    # ── 多记录分组：每组返回 2 条 ──
    result2 = client.search(
        collection_name=collection_name,
        data=[query_vector],
        anns_field="content_vector",
        limit=2,                          # 2 个分组
        group_by_field="is_available",
        group_size=2,                     # 每组返回 2 条
        strict_group_size=True,           # 严格保证每组 2 条（数据不足时可能返回少于 2 条）
        output_fields=["book_id", "title", "is_available"]
    )
    # 最多返回 2×2=4 条记录

    print(f"\n多记录分组搜索结果: 共 {len(result2[0])} 条")
    for hit in result2[0]:
        entity = hit['entity']
        status = "在馆" if entity['is_available'] else "不在馆"
        print(f"  [{status}] {entity['title']}, 相似度: {hit['distance']:.4f}")

2.4 strict_group_size 的区别

# strict_group_size=False（默认）
# 优先保证返回 limit 个分组，每组尽量返回 group_size 条
# 数据分布不均时，某些组可能少于 group_size 条
# 性能更好

# strict_group_size=True
# 严格保证每组返回 group_size 条（数据不足的组除外）
# 数据分布不均时性能可能下降
# 适合对每组数量有严格要求的场景

2.5 支持的索引类型

分组搜索只支持以下索引：FLAT、IVF_FLAT、IVF_SQ8、HNSW、HNSW_PQ、HNSW_PRQ、HNSW_SQ、DISKANN、SPARSE_INVERTED_INDEX

⚠️ 不支持 FLOAT 类型字段分组，如果对 rating（FLOAT）分组会报错，需改用 INT 或 VARCHAR 字段。

三、混合搜索（Hybrid Search）

3.1 核心概念

混合搜索允许同时在多个向量字段上搜索，然后用重排序策略合并结果。

为什么需要混合搜索？

单一向量的局限性：

  • 稠密向量（Dense）：擅长语义理解，但对精确关键词匹配弱
  • 稀疏向量（Sparse）：擅长关键词匹配，但缺乏语义理解

混合搜索结合两者优势，典型组合：

稠密向量（语义）+ 稀疏向量（关键词）→ 更准确的文本检索
文本向量 + 图像向量 → 多模态搜索
多个文本字段向量 → 综合多维度信息

3.2 三步走流程

Step 1: 为每个向量字段创建 AnnSearchRequest
Step 2: 配置重排序策略（RRF 或 Weighted）
Step 3: 调用 hybrid_search() 执行

3.3 代码示例

def hybrid_vector_search_demo():
    """
    混合搜索：同时搜索 content_vector（稠密）和 keyword_vector（稀疏）
    使用 RRF 重排序策略合并结果
    """
    from pymilvus import AnnSearchRequest, Function, FunctionType

    query_vector = [float(random.uniform(-1, 1)) for _ in range(256)]

    # ── Step 1：为每个向量字段创建独立的搜索请求 ──

    # 稠密向量搜索请求（语义相似度）
    content_search_req = AnnSearchRequest(
        data=[query_vector],           # 查询向量，必须是列表
        anns_field="content_vector",   # 目标向量字段
        param={"metric_type": "COSINE", "params": {"nprobe": 10}},
        limit=3                        # 每个子请求返回的候选数
        # expr="is_available == True"  # 可选：子请求也支持过滤条件
    )

    # 稀疏向量搜索请求（关键词匹配）
    # 稀疏向量用字典表示：{维度索引: 权重值}
    keyword_query = {0: 0.8, 10: 0.6, 25: 0.4}
    keyword_search_req = AnnSearchRequest(
        data=[keyword_query],
        anns_field="keyword_vector",
        param={"metric_type": "IP", "params": {}},  # 稀疏向量用 IP
        limit=3
    )

    # ── Step 2：配置重排序策略 ──

    # RRF（互惠排名融合）：不需要手动调权重，适合不确定各路权重时使用
    ranker = Function(
        name="rrf",
        input_field_names=[],          # 必须为空列表，不能填向量字段名
        function_type=FunctionType.RERANK,
        params={
            "reranker": "rrf",
            "k": 100                   # 控制排名衰减速度，默认 100，一般不需要改
        }
    )

    # ── Step 3：执行混合搜索 ──
    result = client.hybrid_search(
        collection_name=collection_name,
        reqs=[content_search_req, keyword_search_req],  # 所有子搜索请求
        ranker=ranker,
        limit=3,                       # 最终返回的总条数
        output_fields=["book_id", "title", "rating"]
        # filter="is_available == True"  # 可选：对最终结果再过滤
    )

    print(f"混合搜索结果: 共 {len(result[0])} 条")
    for i, hit in enumerate(result[0]):
        entity = hit['entity']
        print(f"  {i+1}. ID: {entity['book_id']}, 标题: {entity['title']}, "
              f"评分: {entity['rating']}, 综合得分: {hit['distance']:.4f}")

3.4 两种重排序策略对比

RRF（Reciprocal Rank Fusion，互惠排名融合）

ranker = Function(
    name="rrf",
    input_field_names=[],
    function_type=FunctionType.RERANK,
    params={"reranker": "rrf", "k": 100}
)
# 原理：综合得分 = Σ 1/(k + rank_i)
# 特点：不需要调权重，各路搜索地位平等
# 适用：不确定各路重要性，或各路同等重要时

Weighted（加权融合）

ranker = Function(
    name="weighted",
    input_field_names=[],
    function_type=FunctionType.RERANK,
    params={
        "reranker": "weighted",
        "weights": [0.7, 0.3]   # 对应 reqs 列表中每个请求的权重，总和为 1
    }
)
# 特点：可以手动控制每路搜索的权重
# 适用：明确知道哪路搜索更重要时
# 注意：weights 列表长度必须与 reqs 列表长度一致

3.5 混合搜索 vs 普通搜索

对比项普通搜索 search()混合搜索 hybrid_search()
向量字段数1 个多个
入口函数client.search()client.hybrid_search()
子请求无AnnSearchRequest 列表
重排序无ranker 参数（RRF / Weighted）
过滤条件filter 参数子请求或最终的 filter

3.6 常见坑

# ❌ 错误：input_field_names 填了向量字段名
ranker = Function(
    input_field_names=["content_vector", "keyword_vector"],  # 错！
    ...
)

# ✅ 正确：必须为空列表
ranker = Function(
    input_field_names=[],  # 必须空列表
    ...
)

# ❌ 错误：每个 AnnSearchRequest 的 data 传了多个向量
AnnSearchRequest(data=[vec1, vec2], ...)  # 每个子请求只支持 1 个查询向量

# ✅ 正确
AnnSearchRequest(data=[vec1], ...)
