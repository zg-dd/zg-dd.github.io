一、Milvus 五种查询方法全面对比

Milvus 提供了 5 种查询方法，分为标量查询（3种）和向量搜索（2种）两大类。

1.2 标量查询 vs 向量搜索

标量查询（Get / Query / QueryIterator）：

  • 不涉及向量计算，查询速度快
  • 基于主键索引或标量字段索引
  • 适合精确查找、条件筛选、数据导出
  • 返回结果无相似度排序

向量搜索（Search / SearchIterator）：

  • 涉及向量相似度计算（ANN 搜索）
  • 基于向量索引（HNSW、IVF 等）
  • 适合语义搜索、推荐系统、相似内容查找
  • 返回结果按相似度排序，包含 distance 字段

1.3 方法选择决策树

需要查询数据？
├── 基于向量相似度？
│   ├── 是 → 向量搜索
│   │   ├── 结果数量 < 16384？
│   │   │   └── 是 → 使用 Search
│   │   └── 结果数量 > 16384？
│   │       └── 是 → 使用 SearchIterator
│   └── 否 → 标量查询
│       ├── 已知主键？
│       │   └── 是 → 使用 Get（最快）
│       └── 否 → 条件筛选，判断数据量
│           ├── 结果数量 < 1万条 → 使用 Query
│           └── 结果数量 > 1万条 → 使用 QueryIterator

标量查询是指不涉及向量相似度计算的查询方式，仅通过主键或标量字段条件进行数据检索。本节重点介绍 Get、Query、QueryIterator 三种标量查询方法。

二、Get 方法：按主键查询

2.1 核心特点

  • 最快的查询方式：直接通过主键索引查找，时间复杂度 O(1)
  • 不涉及向量计算：纯标量字段查询
  • 精确匹配：必须提供准确的主键值

2.2 代码示例

from pymilvus import MilvusClient

client = MilvusClient(uri="http://localhost:19530")

# 按主键获取实体
result = client.get(
    collection_name="book_collection",
    ids=[1, 5, 10],  # 主键列表
    output_fields=["book_id", "title", "isbn", "rating"]
)

# 遍历结果
for record in result:
    print(f"ID: {record['book_id']}, 标题: {record['title']}")

2.3 参数说明

参数类型必需说明
collection_namestr是集合名称
idsList是主键列表
output_fieldsList[str]否返回的字段列表，默认只返回主键
partition_namesList[str]否限定查询的分区

2.4 返回结果结构

result = [
    {
        'book_id': 1,
        'title': 'Python编程',
        'isbn': '978-1234567890',
        'rating': 4.5
    },
    {
        'book_id': 5,
        'title': 'AI实战',
        'isbn': '978-0987654321',
        'rating': 4.8
    }
]

三、Query 方法：条件过滤查询

3.1 核心特点

  • 灵活的条件筛选：支持丰富的过滤表达式
  • 可控的返回数量：通过 limit 参数限制结果数
  • 不涉及向量计算：纯标量字段查询

3.2 基础用法

# 简单条件查询
result = client.query(
    collection_name="book_collection",
    filter="is_available == True",
    output_fields=["book_id", "title", "is_available"],
    limit=10
)

# 复合条件查询
result = client.query(
    collection_name="book_collection",
    filter="is_available == True and rating >= 4.0",
    output_fields=["book_id", "title", "rating"],
    limit=5
)

3.3 参数说明

参数类型必需说明
collection_namestr是集合名称
filterstr是过滤表达式（空字符串表示查询全部）
output_fieldsList[str]否返回的字段列表
limitint否返回结果数量上限，默认 16384
partition_namesList[str]否限定查询的分区
filter_paramsdict否过滤模板参数（见 3.6 节）

四、过滤表达式详解

4.1 比较操作符

# 等于
filter = "is_available == True"
filter = "rating == 4.5"

# 不等于
filter = "rating != 3.0"

# 大于/小于
filter = "rating > 4.0"
filter = "price < 100.0"
filter = "page_count >= 200"
filter = "rating <= 5.0"

4.2 范围操作符

# IN 操作符
filter = "rating in [4.0, 4.5, 5.0]"
filter = "book_id in [1, 2, 3, 5, 8]"

# LIKE 操作符（模糊匹配）
filter = 'title like "%Python%"'    # 包含 "Python"
filter = 'title like "Python%"'     # 以 "Python" 开头
filter = 'title like "%编程"'        # 以 "编程" 结尾

4.3 逻辑操作符

# AND（与）
filter = "is_available == True and rating >= 4.0"

# OR（或）
filter = 'title like "%Python%" or title like "%AI%"'

# NOT（非）
filter = "not is_available == True"

# 复杂组合
filter = """
(is_available == True and rating >= 4.0) and
(price < 150.0 or page_count > 300)
"""

4.4 空值判断

# 非空
filter = "details is not null"

# 空值
filter = "description is null"

4.5 算术操作符

# 取模（查找偶数 ID）
filter = "book_id % 2 == 0"

# 乘法
filter = "price * 0.8 < 100"

# 除法
filter = "page_count / 100 > 2"

# 加减
filter = "rating + 0.5 >= 5.0"

4.6 过滤模板（Filter Templates）

为什么需要过滤模板？

  • 提高处理 CJK（中日韩）字符时的性能
  • 增强代码可读性和安全性
  • 查询引擎可缓存解析后的表达式结构

基本用法：

# 定义模板（使用 {param_name} 占位符）
template_filter = "rating >= {min_rating} and is_available == {available}"

# 传递参数
template_params = {
    "min_rating": 4.0,
    "available": True
}

# 执行查询
result = client.query(
    collection_name="book_collection",
    filter=template_filter,
    filter_params=template_params,
    output_fields=["book_id", "title", "rating"],
    limit=10
)

数组 IN 操作模板：

# 模板定义
in_filter = "rating in {ratings}"

# 参数传递
in_params = {"ratings": [4.0, 4.5, 5.0]}

result = client.query(
    collection_name="book_collection",
    filter=in_filter,
    filter_params=in_params,
    output_fields=["book_id", "title", "rating"],
    limit=10
)

注意事项：

  • 占位符名称必须与 filter_params 中的键名完全匹配
  • 确保参数值的数据类型与字段类型匹配
  • 需要 Milvus 2.4+ 版本支持

五、QueryIterator 方法：分页查询

5.1 核心特点

  • 内存友好：流式处理，不会一次性加载全部结果
  • 适合大数据量：通过分批获取避免内存溢出
  • 不涉及向量计算：纯标量字段查询

5.2 代码示例

# 创建查询迭代器
iterator = client.query_iterator(
    collection_name="book_collection",
    filter="is_available == True",
    output_fields=["book_id", "title", "is_available"],
    batch_size=50  # 每批返回 50 条
)

# 分批获取结果
all_results = []
while True:
    batch_result = iterator.next()
    if not batch_result:
        break

    all_results.extend(batch_result)
    print(f"本批返回 {len(batch_result)} 条记录")

print(f"总共获取 {len(all_results)} 条记录")

5.3 参数说明

参数类型必需说明
collection_namestr是集合名称
filterstr是过滤表达式
output_fieldsList[str]否返回的字段列表
batch_sizeint是每批返回的记录数
partition_namesList[str]否限定查询的分区

5.4 使用场景

  • 需要导出大量数据
  • 需要遍历集合中的所有匹配记录
  • 内存受限的环境
  • 需要实时处理查询结果（流式处理）

六、特殊查询技巧

6.1 统计实体数量

# 统计集合总数
result = client.query(
    collection_name="book_collection",
    filter="",
    output_fields=["count(*)"]
)
print(result)  # [{'count(*)': 2000}]

# 统计满足条件的数量
result = client.query(
    collection_name="book_collection",
    filter="rating >= 4.5",
    output_fields=["count(*)"]
)

6.2 查询所有数据

# 空过滤表达式查询全部
result = client.query(
    collection_name="book_collection",
    filter="",
    output_fields=["book_id", "title"],
    limit=100
)

6.3 分区查询

# 限定在特定分区查询
result = client.query(
    collection_name="book_collection",
    filter="rating >= 4.0",
    partition_names=["partition_2024"],
    output_fields=["book_id", "title"]
)

七、方法选择指南

7.1 决策树

需要查询数据？
├── 已知主键？
│   └── 是 → 使用 Get（最快）
└── 否 → 需要条件筛选
    ├── 数据量小（< 1万条）？
    │   └── 是 → 使用 Query
    └── 数据量大（> 1万条）？
        └── 是 → 使用 QueryIterator

7.2 性能对比

方法查询速度内存占用适用数据量
Get最快低任意
Query快中< 1万条
QueryIterator中低（流式）任意

八、常见问题

8.1 Query 返回结果为空

可能原因：

  • 过滤条件过于严格
  • 集合未加载到内存
  • 字段名拼写错误

解决方法：

# 1. 检查集合加载状态
load_state = client.get_load_state(collection_name="book_collection")
print(load_state)

# 2. 简化过滤条件测试
result = client.query(
    collection_name="book_collection",
    filter="",  # 空过滤查询全部
    limit=1
)

8.2 QueryIterator 内存占用过高

原因： batch_size 设置过大

解决方法：

# 减小 batch_size
iterator = client.query_iterator(
    collection_name="book_collection",
    filter="is_available == True",
    batch_size=10  # 从 100 降到 10
)

8.3 过滤表达式语法错误

常见错误：

# ❌ 错误：字符串未加引号
filter = "title == Python"

# ✅ 正确
filter = 'title == "Python"'

# ❌ 错误：LIKE 通配符位置错误
filter = 'title like "Python*"'

# ✅ 正确
filter = 'title like "Python%"'

  • 逻辑：and, or, not
