1. What is a Struct

A struct is an important composite data type in Go language that can combine multiple data of different types together to form a meaningful whole. A struct can be understood as an abstract representation of things in the real world.

1.1 Basic Concepts of Structs

Imagine we want to describe a student’s information:

  • Name: string type
  • Age: integer type
  • Score: floating-point type
  • Enrolled status: boolean type

Without structs, we need to define multiple variables separately:

var name string = "Zhang San"
var age int = 18
var score float64 = 95.5
var isEnrolled bool = true

This is very inconvenient to manage and use. Structs can organize these related data together:

type Student struct {
    Name      string
    Age       int
    Score     float64
    IsEnrolled bool
}

1.2 Why Do We Need Structs

Main advantages of structs:

  1. Data Aggregation: Organize related data together
  2. Code Clarity: Improve code readability and maintainability
  3. Type Safety: Create specialized data types
  4. Object-Oriented: Lay foundation for subsequent methods and interfaces

2. Struct Definition and Declaration

2.1 Basic Definition Syntax

// Basic syntax for struct definition
type struct_name struct {
    field_name1 data_type1
    field_name2 data_type2
    field_name3 data_type3
    // ... more fields
}

2.3 Anonymous Structs

package main

import "fmt"

func main() {
    // Anonymous struct definition and usage
    person := struct {
        Name string
        Age  int
    }{
        Name: "Li Si",
        Age:  25,
    }
    
    fmt.Printf("Anonymous struct: %+v\n", person)
    fmt.Printf("Name: %s, Age: %d\n", person.Name, person.Age)
    
    // Anonymous struct slice
    people := []struct {
        Name string
        City string
    }{
        {"Wang Wu", "Beijing"},
        {"Zhao Liu", "Shanghai"},
        {"Sun Qi", "Guangzhou"},
    }
    
    fmt.Println("People list:")
    for _, p := range people {
        fmt.Printf("Name: %s, City: %s\n", p.Name, p.City)
    }
}

Execution result:

Anonymous struct: {Name:Li Si Age:25}
Name: Li Si, Age: 25
People list:
Name: Wang Wu, City: Beijing
Name: Zhao Liu, City: Shanghai
Name: Sun Qi, City: Guangzhou

3. Struct Instantiation

3.1 Various Creation Methods

package main

import "fmt"

type Student struct {
    Name  string
    Age   int
    Score float64
}

func main() {
    // Method 1: Complete field initialization
    student1 := Student{
        Name:  "Zhang San",
        Age:   18,
        Score: 95.5,
    }
    
    // Method 2: Initialize by field order (must specify all fields)
    student2 := Student{"Li Si", 19, 88.0}
    
    // Method 3: Partial field initialization (must specify field names)
    student3 := Student{
        Name: "Wang Wu",
        Age:  20,
        // Score uses default value 0.0
    }
    
    // Method 4: Use new function to create (all fields have zero values)
    student4 := new(Student)
    student4.Name = "Zhao Liu"
    student4.Age = 21
    student4.Score = 92.5
    
    // Method 5: Use & to create and initialize
    student5 := &Student{
        Name:  "Sun Qi",
        Age:   17,
        Score: 87.5,
    }
    
    fmt.Printf("Student 1: %+v\n", student1)
    fmt.Printf("Student 2: %+v\n", student2)
    fmt.Printf("Student 3: %+v\n", student3)
    fmt.Printf("Student 4: %+v\n", *student4)
    fmt.Printf("Student 5: %+v\n", *student5)
}

Execution result:

Student 1: {Name:Zhang San Age:18 Score:95.5}
Student 2: {Name:Li Si Age:19 Score:88}
Student 3: {Name:Wang Wu Age:20 Score:0}
Student 4: {Name:Zhao Liu Age:21 Score:92.5}
Student 5: {Name:Sun Qi Age:17 Score:87.5}

type Person struct {
    Name     string
    Age      int
    Height   float64
    IsActive bool
    Scores   []int
    Info     map[string]string
}

    // Declare struct variable without initialization
    var person Person
    
    fmt.Printf("Struct zero value: %+v\n", person)
    fmt.Printf("String zero value: %q\n", person.Name)
    fmt.Printf("Integer zero value: %d\n", person.Age)
    fmt.Printf("Float zero value: %f\n", person.Height)
    fmt.Printf("Boolean zero value: %t\n", person.IsActive)

    fmt.Printf("Map zero value: %v, is nil: %t\n", person.Info, person.Info == nil)
}

String zero value: ""
Integer zero value: 0

4. Struct Field Access and Modification

4.1 Field Access

package main

import "fmt"

type Employee struct {
    Name       string
    Age        int
    Salary     float64
    Department string
}

func main() {
    // Create employee instance
    emp := Employee{
        Name:       "Zhang San",
        Age:        28,
        Salary:     15000.0,
        Department: "Technology Department",
    }
    
    // Access fields
    fmt.Printf("Employee name: %s\n", emp.Name)
    fmt.Printf("Employee age: %d\n", emp.Age)
    fmt.Printf("Employee salary: %.2f\n", emp.Salary)

    fmt.Printf("Employee age: %d\n", emp.Age)
    fmt.Printf("Employee salary: %.2f\n", emp.Salary)
    fmt.Printf("Department: %s\n", emp.Department)
}

Execution result:

Employee age: 28
Employee salary: 15000.00
Department: Technology Department

Information after modification:

Employee age: 29

4.2 Pointer Access to Struct Fields

package main

import "fmt"

type Product struct {
    Name  string
    Price float64
    Stock int
}

func main() {
    // Ordinary struct variable
    product1 := Product{Name: "Phone", Price: 3999.0, Stock: 100}
    
    // Struct pointer
    product2 := &Product{Name: "Computer", Price: 8999.0, Stock: 50}
    
    // Access method 1: Direct access through variable name
    fmt.Printf("Product 1 - Name: %s, Price: %.2f, Stock: %d\n", 
               product1.Name, product1.Price, product1.Stock)
    
    // Access method 2: Access through pointer (Go automatically dereferences)
    fmt.Printf("Product 2 - Name: %s, Price: %.2f, Stock: %d\n", 
               product2.Name, product2.Price, product2.Stock)
    
    // Access method 3: Explicit dereferencing
    fmt.Printf("Product 2 explicit dereference - Name: %s, Price: %.2f, Stock: %d\n", 
               (*product2).Name, (*product2).Price, (*product2).Stock)
    
    // Modify values
    product1.Price = 3599.0
    product2.Stock = 75  // Equivalent to (*product2).Stock = 75
    
    fmt.Printf("\nAfter modification:\n")
    fmt.Printf("Product 1 price: %.2f\n", product1.Price)
    fmt.Printf("Product 2 stock: %d\n", product2.Stock)
}

Execution result:

Product 1 - Name: Phone, Price: 3999.00, Stock: 100
Product 2 - Name: Computer, Price: 8999.00, Stock: 50
Product 2 explicit dereference - Name: Computer, Price: 8999.00, Stock: 50

After modification:
Product 1 price: 3599.00
Product 2 stock: 75

5. Struct Nesting

5.1 Basic Nesting

package main

import "fmt"

// Address struct
type Address struct {
    Province string
    City     string
    Street   string
    ZipCode  string
}

// Company struct
type Company struct {
    Name    string
    Address Address  // Nested Address struct
}

// Employee struct
type Employee struct {
    Name    string
    Age     int
    Company Company  // Nested Company struct
}

func main() {
    // Create nested struct
    employee := Employee{
        Name: "Zhang San",
        Age:  30,
        Company: Company{
            Name: "Technology Co., Ltd.",
            Address: Address{
                Province: "Beijing",
                City:     "Beijing",
                Street:   "Zhongguancun Street 1",
                ZipCode:  "100080",
            },
        },
    }
    
    // Access nested fields
    fmt.Printf("Employee name: %s\n", employee.Name)
    fmt.Printf("Employee age: %d\n", employee.Age)
    fmt.Printf("Company name: %s\n", employee.Company.Name)
    fmt.Printf("Company province: %s\n", employee.Company.Address.Province)
    fmt.Printf("Company city: %s\n", employee.Company.Address.City)
    fmt.Printf("Company street: %s\n", employee.Company.Address.Street)
    fmt.Printf("Zip code: %s\n", employee.Company.Address.ZipCode)
    
    // Modify nested fields
    employee.Company.Address.Street = "Zhongguancun Street 2"
    fmt.Printf("\nStreet after modification: %s\n", employee.Company.Address.Street)
}

Execution result:

Employee name: Zhang San
Employee age: 30
Company name: Technology Co., Ltd.
Company province: Beijing
Company city: Beijing
Company street: Zhongguancun Street 1
Zip code: 100080

Street after modification: Zhongguancun Street 2

5.2 Anonymous Nesting (Anonymous Fields)

package main

import "fmt"

type Person struct {
    Name string
    Age  int
}

type Student struct {
    Person      // Anonymous nested Person struct
    School string
    Grade  int
}

type Teacher struct {
    Person      // Anonymous nested Person struct
    Subject string
    Salary  float64
}

func main() {
    // Create student
    student := Student{
        Person: Person{Name: "Xiao Ming", Age: 18},
        School: "No.1 Middle School",
        Grade:  12,
    }
    
    // Create teacher
    teacher := Teacher{
        Person:  Person{Name: "Teacher Wang", Age: 35},
        Subject: "Mathematics",
        Salary:  8000.0,
    }
    
    // Access promoted fields (can directly access nested struct fields)
    fmt.Printf("Student name: %s\n", student.Name)  // Equivalent to student.Person.Name
    fmt.Printf("Student age: %d\n", student.Age)   // Equivalent to student.Person.Age
    fmt.Printf("School: %s\n", student.School)
    fmt.Printf("Grade: %d\n", student.Grade)
    
    fmt.Printf("\nTeacher name: %s\n", teacher.Name)  // Equivalent to teacher.Person.Name
    fmt.Printf("Teacher age: %d\n", teacher.Age)     // Equivalent to teacher.Person.Age
    fmt.Printf("Subject: %s\n", teacher.Subject)
    fmt.Printf("Salary: %.2f\n", teacher.Salary)
    
    // Modify promoted fields
    student.Age = 19
    teacher.Name = "Teacher Li"
    
    fmt.Printf("\nStudent age after modification: %d\n", student.Age)
    fmt.Printf("Teacher name after modification: %s\n", teacher.Name)
}

Execution result:

Student name: Xiao Ming
Student age: 18
School: No.1 Middle School
Grade: 12

Teacher name: Teacher Wang
Teacher age: 35
Subject: Mathematics
Salary: 8000.00

Student age after modification: 19
Teacher name after modification: Teacher Li

6. Struct Tags

6.1 Basic Tag Usage

package main

import (
    "encoding/json"
    "fmt"
)

type User struct {
    ID       int    `json:"id"`
    UserName string `json:"username"`
    Email    string `json:"email"`
    Password string `json:"-"`              // - means ignore this field
    Active   bool   `json:"active,omitempty"` // omitempty means ignore when value is empty
}

func main() {
    user := User{
        ID:       1,
        UserName: "zhangsan",
        Email:    "zhangsan@example.com",
        Password: "123456",
        Active:   true,
    }
    
    // Serialize to JSON
    jsonData, err := json.Marshal(user)
    if err != nil {
        fmt.Printf("Serialization error: %v\n", err)
        return
    }
    
    fmt.Printf("JSON data: %s\n", jsonData)
    
    // Deserialize
    var newUser User
    jsonStr := `{"id":2,"username":"lisi","email":"lisi@example.com","active":false}`
    err = json.Unmarshal([]byte(jsonStr), &newUser)
    if err != nil {
        fmt.Printf("Deserialization error: %v\n", err)
        return
    }
    
    fmt.Printf("Deserialization result: %+v\n", newUser)
}

Execution result:

JSON data: {"id":1,"username":"zhangsan","email":"zhangsan@example.com","active":true}
Deserialization result: {ID:2 UserName:lisi Email:lisi@example.com Password: Active:false}

6.2 Multiple Tags Example

package main

    "encoding/json"
    "fmt"
)

type Product struct {
    ID          int     `json:"id" db:"product_id"`
    Name        string  `json:"name" db:"product_name"`
    Price       float64 `json:"price" db:"price"`
    Description string  `json:"description,omitempty" db:"description"`
    Category    string  `json:"category" db:"category"`
}

func main() {
    products := []Product{
        {
            ID:          1,
            Name:        "iPhone 15",
            Price:       5999.0,
            Description: "Latest Apple phone",
            Category:    "Electronics",
        },
        {
            ID:          2,
            Name:        "MacBook Pro",
            Price:       12999.0,
            // Description is empty, will be ignored by omitempty

        },
    }
    
    // JSON serialization
    jsonData, _ := json.MarshalIndent(products, "", "  ")
    fmt.Printf("JSON formatted data:\n%s\n", jsonData)
}

7. Struct Comparison

7.1 Struct Equality Comparison

package main

import "fmt"

type Point struct {
    X, Y int
}

type Person struct {
    Name string
    Age  int
    Data []int  // Slice field
}

func main() {
    // Comparable structs
    p1 := Point{1, 2}
    p2 := Point{1, 2}
    p3 := Point{3, 4}
    
    fmt.Printf("p1 == p2: %t\n", p1 == p2)
    fmt.Printf("p1 == p3: %t\n", p1 == p3)
    
    // Structs containing non-comparable fields
    person1 := Person{Name: "Zhang San", Age: 20, Data: []int{1, 2, 3}}
    person2 := Person{Name: "Zhang San", Age: 20, Data: []int{1, 2, 3}}
    
    // The following code would cause compilation error because it contains slice fields
    // fmt.Printf("person1 == person2: %t\n", person1 == person2)
    
    // Manual comparison
    equal := person1.Name == person2.Name && 
             person1.Age == person2.Age &&
             len(person1.Data) == len(person2.Data)
    
    if equal {
        for i := range person1.Data {
            if person1.Data[i] != person2.Data[i] {
                equal = false
                break
            }
        }
    }
    
    fmt.Printf("Manual comparison result: %t\n", equal)
}

Execution result:

p1 == p2: true
p1 == p3: false
Manual comparison result: true

8. Struct Factory Functions

8.1 Constructor Functions for Creating Structs

package main

import (
    "fmt"
    "time"
)

type User struct {
    ID        int
    Username  string
    Email     string
    CreatedAt time.Time
    IsActive  bool
}

// Constructor 1: Create ordinary user
func NewUser(username, email string) *User {
    return &User{
        Username:  username,
        Email:     email,
        CreatedAt: time.Now(),
        IsActive:  true,
    }
}

// Constructor 2: Create admin user
func NewAdminUser(username, email string) *User {
    user := NewUser(username, email)
    user.ID = -1  // Set admin ID to special value
    return user
}

// Constructor 3: Constructor with validation
func NewUserWithValidation(username, email string) (*User, error) {
    if len(username) < 3 {
        return nil, fmt.Errorf("username length cannot be less than 3 characters")
    }
    if len(email) < 5 || !contains(email, "@") {
        return nil, fmt.Errorf("incorrect email format")
    }
    
    return &User{
        Username:  username,
        Email:     email,
        CreatedAt: time.Now(),
        IsActive:  true,
    }, nil
}

// Helper function
func contains(s, substr string) bool {
    for i := 0; i <= len(s)-len(substr); i++ {
        if s[i:i+len(substr)] == substr {
            return true
        }
    }
    return false
}

func main() {
    // Create users using constructors
    user1 := NewUser("zhangsan", "zhangsan@example.com")
    fmt.Printf("Ordinary user: %+v\n", *user1)
    
    admin := NewAdminUser("admin", "admin@example.com")
    fmt.Printf("Admin user: %+v\n", *admin)
    
    // Constructor with validation
    user2, err := NewUserWithValidation("li", "invalid-email")
    if err != nil {
        fmt.Printf("User creation failed: %v\n", err)
    }
    
    user3, err := NewUserWithValidation("lisi", "lisi@example.com")
    if err != nil {
        fmt.Printf("User creation failed: %v\n", err)
    } else {
        fmt.Printf("Validated user: %+v\n", *user3)
    }
}

Execution result:

Ordinary user: {ID:0 Username:zhangsan Email:zhangsan@example.com CreatedAt:2024-01-15 10:30:45.123456789 +0800 CST IsActive:true}
Admin user: {ID:-1 Username:admin Email:admin@example.com CreatedAt:2024-01-15 10:30:45.123456789 +0800 CST IsActive:true}
User creation failed: username length cannot be less than 3 characters
Validated user: {ID:0 Username:lisi Email:lisi@example.com CreatedAt:2024-01-15 10:30:45.123456789 +0800 CST IsActive:true}

9. Structs and Functions

9.1 Structs as Function Parameters

package main

import "fmt"

type Rectangle struct {
    Width  float64
    Height float64
}

// Pass by value
func calculateArea(rect Rectangle) float64 {
    return rect.Width * rect.Height
}

// Pass by pointer (can modify original struct)
func scaleRectangle(rect *Rectangle, factor float64) {
    rect.Width *= factor
    rect.Height *= factor
}

// Return struct
func createSquare(side float64) Rectangle {
    return Rectangle{Width: side, Height: side}
}

func main() {
    // Create rectangle
    rect := Rectangle{Width: 10, Height: 5}
    fmt.Printf("Original rectangle: %+v\n", rect)
    
    // Calculate area by value passing
    area := calculateArea(rect)
    fmt.Printf("Area: %.2f\n", area)
    fmt.Printf("Rectangle after calculation: %+v\n", rect)  // Original rectangle unchanged
    
    // Scale by pointer passing
    scaleRectangle(&rect, 2.0)
    fmt.Printf("Rectangle after scaling: %+v\n", rect)
    fmt.Printf("Area after scaling: %.2f\n", calculateArea(rect))
    
    // Create square
    square := createSquare(8)
    fmt.Printf("Created square: %+v\n", square)
    fmt.Printf("Square area: %.2f\n", calculateArea(square))
}

Execution result:

Original rectangle: {Width:10 Height:5}
Area: 50.00
Rectangle after calculation: {Width:10 Height:5}
Rectangle after scaling: {Width:20 Height:10}
Area after scaling: 200.00
Created square: {Width:8 Height:8}
Square area: 64.00

10. Summary

Structs are important tools in Go language for organizing and managing related data:

Key Features:

  • Combine multiple related data fields together
  • Can nest other structs
  • Support tags for operations like serialization
  • Constructor functions can be defined to standardize creation process

Usage Scenarios:

  1. Represent real-world entities (users, products, orders, etc.)
  2. Organize configuration information
  3. Pass complex data as function parameters
  4. Foundation for object-oriented programming implementation

Best Practices:

  • Use constructor functions to create struct instances
  • Add tags to important structs
  • Reasonably use pointer passing to avoid unnecessary copying
  • Pay attention to struct comparison issues with non-comparable fields

Structs are the foundation of object-oriented programming in Go language. Mastering struct usage is very important for subsequent learning of methods and interfaces.
