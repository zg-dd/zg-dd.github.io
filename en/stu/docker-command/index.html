Common Docker & Docker Compose Commands

1. Image management

Images are like installation packages or templates.

CommandDescriptionExample
docker imagesdocker image lsList local imagesdocker images
docker search <name>Search images from registrydocker search nginx
docker pull <image>:<tag>Pull image (defaults to latest)docker pull nginx:alpine
docker rmi <image>Remove local imagedocker rmi nginx
docker image pruneRemove dangling imagesdocker image prune
docker build -t <tag> .Build image from Dockerfiledocker build -t my-app:1.0 .

2. Container management

Containers are runtime instances created from images.

CommandDescriptionExample
docker run [options] <image>Create and start a containerdocker run -d -p 80:80 nginx
docker start <name/ID>Start a stopped containerdocker start my_nginx
docker stop <name/ID>Stop a running container (graceful)docker stop my_nginx
docker restart <name/ID>Restart containerdocker restart my_nginx
docker rm <name/ID>Remove a stopped containerdocker rm my_nginx
docker container pruneRemove all stopped containersdocker container prune
docker psList running containersdocker ps
docker ps -aList all containersdocker ps -a
docker logs <name/ID>View container logsdocker logs my_nginx
docker logs -f <name/ID>Follow logs in real timedocker logs -f my_nginx
docker exec -it <name/ID> <cmd>Run a command inside containerdocker exec -it my_nginx /bin/bash

3. Network management

CommandDescriptionExample
docker network lsList networksdocker network ls
docker network create <name>Create a networkdocker network create my_net
docker network inspect <name>Inspect a networkdocker network inspect bridge

4. Volume management

Volumes store persistent data.

CommandDescriptionExample
docker volume lsList volumesdocker volume ls
docker volume create <name>Create a volumedocker volume create my_data
docker volume rm <name>Remove a volumedocker volume rm my_data
docker volume pruneRemove unused volumesdocker volume prune

5. System and info

CommandDescriptionExample
docker infoShow Docker system infodocker info
docker versionShow Docker versiondocker version
docker statsLive resource usagedocker stats
docker system dfDisk usagedocker system df

6. Docker Compose commands

Manage multi-container apps.

CommandDescriptionExample
docker compose upCreate and start servicesdocker compose up
docker compose up -dStart services in backgrounddocker compose up -d
docker compose downStop and remove servicesdocker compose down
docker compose psList project containersdocker compose ps
docker compose logsView logsdocker compose logs
docker compose logs -f <service>Follow logs for a servicedocker compose logs -f web
docker compose exec <service> <cmd>Run a command in service containerdocker compose exec db mysql -u root -p

Common flags

FlagLongMeaningExample
-d--detachRun in backgrounddocker run -d nginx
-it-i -tInteractive TTYdocker run -it ubuntu /bin/bash
-p--publishPort mappingdocker run -p 8080:80 nginx
-v--volumeMount volume or dirdocker run -v /host/data:/container/data nginx
-e--envSet env vardocker run -e MY_VAR=value nginx
--nameSet container namedocker run --name my_nginx nginx
--rmRemove container on exitdocker run --rm alpine echo "hello"

üöÄ Most-used commands

  1. Check status: docker ps (containers), docker images (images)
  2. Pull image: docker pull <name>
  3. Run container: docker run -d -p ... --name ... <image>
  4. Stop container: docker stop <name>
  5. Enter container: docker exec -it <name> /bin/bash
  6. View logs: docker logs -f <name>
  7. Remove container: docker rm <name>
  8. Remove image: docker rmi <name>

Tip: Save this list as a daily quick reference.
