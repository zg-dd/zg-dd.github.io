Regex Concepts

  1. [] Use [] to match a single character inside the brackets. For example, [abc] matches any one of a, b, or c. In the string “abcdabcdbd” it matches a, b, c, a, b, c, b. It matches one character at a time and does not merge them. [abc]+ means one or more occurrences of any character in a, b, c. In the same string it matches abc, abc, b. In other words, it matches any of a, b, c one or more times, and consecutive matches are grouped as one string.

  2. \ \ is the escape character. Common escapes include \n (newline), \t (tab), \r (carriage return), \b (backspace), \f (form feed), \v (vertical tab), ' (single quote), " (double quote), and \ (backslash). Regex has many other symbols; the image below lists them. Important: In regex, some escapes like \b, \f, \n, \t, \r, ' can be written directly, e.g. regex="\n"; that matches a newline. regex="\\n" also matches a newline. But for symbols like \w, \W, \B, \S, \s, you must write regex="\\w" to be valid; otherwise it fails. When you copy these escapes, \ is often automatically doubled. So I recommend using \\ when you need a literal backslash in regex. Note: Not every escape needs to be doubled. For example, to match a double quote, you should not write regex="\"". That would be interpreted incorrectly. Use regex="\\"". Using regex=""" also works. If you want to match a backslash itself, in Java you would write regex=" \\\\ “ (four backslashes). The first two represent the escape for , and the last two represent another literal \ (this is my understanding; the expression is correct). By the same logic, to match \, you would write regex=” \\\\\\\ “.

  3. | This symbol means OR. For x|y, it matches x or y. The | operator has higher precedence than x,y. For regex = “[a][b]|[c][d]”, it matches “ab” or “cd”, not “a(b|c)d”.

  4. ?<= and ?= These are lookaround tokens. They match a symbol without including it in the result. Example: to extract the text inside double quotes, for String str = “he says,\“Hello,Java!\” “; You can use: regex = “(?<=\")[^\"]+(?=\")”. Here, (?<=\”) matches a quote but does not include it, [^\”]+ matches one or more non-quote characters, and (?=\”) matches the closing quote without including it. These two are usually used together: one before and one after.

Note: The single and double quotes above are English quotes. Some symbols may appear as Chinese punctuation due to escaping; keep that in mind when reading. That is all for now; I will add more later if I find new points.
