1. What are Methods

Methods are functions associated with specific types in Go language. If functions are independent functional units, then methods are special functions that “belong” to a particular type. Methods give data types behavioral capabilities, which is an important feature of object-oriented programming.

1.1 Difference Between Methods and Functions

Let’s understand this through a simple example:

// Regular function
func add(a, b int) int {
    return a + b
}

// Method (syntax will be explained in detail later)
// func (receiver) methodName(parameters) returnType { function body }

Main differences:

  • Function: Exists independently, called by function name
  • Method: Binds to a specific type, called by “type.methodName”

1.2 Why Do We Need Methods

Main values of methods:

  1. Encapsulation: Organize data and methods that operate on data together
  2. Readability: student.GetName() is more intuitive than getName(student)
  3. Object-oriented: Lay foundation for subsequent interface implementation
  4. Code organization: Better organization of related functionalities

2. Basic Method Syntax

2.1 Method Definition Syntax

func (receiver) methodName(parameterList) (returnValueList) {
    // method body
}

2.2 Two Forms of Receivers

Value Receiver

func (t MyType) MethodName(parameters) returnType {
    // t is a copy of MyType type
    // Modifying t won't affect original data
}

Pointer Receiver

func (t *MyType) MethodName(parameters) returnType {
    // t is a pointer to MyType type
    // Modifying t will affect original data
}

2.3 Simple Example

package main

import "fmt"

// Define a rectangle struct
type Rectangle struct {
    Width  float64
    Height float64
}

// Value receiver method: calculate area
func (r Rectangle) Area() float64 {
    return r.Width * r.Height
}

// Value receiver method: calculate perimeter
func (r Rectangle) Perimeter() float64 {
    return 2 * (r.Width + r.Height)
}

// Pointer receiver method: scale rectangle
func (r *Rectangle) Scale(factor float64) {
    r.Width *= factor
    r.Height *= factor
}

// Pointer receiver method: set dimensions
func (r *Rectangle) SetDimensions(width, height float64) {
    r.Width = width
    r.Height = height
}

func main() {
    // Create rectangle instance
    rect := Rectangle{Width: 10, Height: 5}
    
    fmt.Printf("Original rectangle: width=%.2f, height=%.2f\n", rect.Width, rect.Height)
    fmt.Printf("Area: %.2f\n", rect.Area())
    fmt.Printf("Perimeter: %.2f\n", rect.Perimeter())
    
    // Use pointer method to modify data
    rect.Scale(2.0)
    fmt.Printf("\nRectangle after scaling: width=%.2f, height=%.2f\n", rect.Width, rect.Height)
    fmt.Printf("New area: %.2f\n", rect.Area())
    
    rect.SetDimensions(15, 8)
    fmt.Printf("\nAfter resetting: width=%.2f, height=%.2f\n", rect.Width, rect.Height)
    fmt.Printf("Final area: %.2f\n", rect.Area())
}

Execution result:

Original rectangle: width=10.00, height=5.00
Area: 50.00
Perimeter: 30.00

Rectangle after scaling: width=20.00, height=10.00
New area: 200.00

After resetting: width=15.00, height=8.00
Final area: 120.00

3. In-depth Understanding of Receivers

3.1 Value Receiver vs Pointer Receiver

package main

import "fmt"

type Counter struct {
    Count int
}

// Value receiver method
func (c Counter) GetValue() int {
    return c.Count
}

// Value receiver method (won't modify original value)
func (c Counter) IncrementByValue() {
    c.Count++  // This modifies the copy
    fmt.Printf("Value receiver internal Count: %d\n", c.Count)
}

// Pointer receiver method
func (c *Counter) GetPointerValue() int {
    return c.Count
}

// Pointer receiver method (will modify original value)
func (c *Counter) IncrementByPointer() {
    c.Count++  // This modifies the original value
    fmt.Printf("Pointer receiver internal Count: %d\n", c.Count)
}

func main() {
    counter := Counter{Count: 10}
    
    fmt.Printf("Initial value: %d\n", counter.Count)
    
    // Value receiver call
    counter.IncrementByValue()
    fmt.Printf("After value receiver call: %d\n", counter.Count)
    
    // Pointer receiver call
    counter.IncrementByPointer()
    fmt.Printf("After pointer receiver call: %d\n", counter.Count)
    
    // Verify return values
    fmt.Printf("GetValue(): %d\n", counter.GetValue())
    fmt.Printf("GetPointerValue(): %d\n", counter.GetPointerValue())
}

Execution result:

Initial value: 10
Value receiver internal Count: 11
Value receiver call after: 10
Pointer receiver internal Count: 11
Pointer receiver call after: 11
GetValue(): 11
GetPointerValue(): 11

3.2 Receiver Selection Principles

When to use value receivers: ‚úÖ Small data types (basic types, small structs) ‚úÖ Methods don’t need to modify receiver data ‚úÖ Concurrency safety required (value passing is safe)

When to use pointer receivers: ‚úÖ Methods need to modify receiver data ‚úÖ Large data types (avoid copy overhead) ‚úÖ Consistency consideration (all methods of type use pointer receivers)

package main

import "fmt"

type SmallData struct {
    Value int
}

type LargeData struct {
    Values [1000]int  // Large array
    Name   string
    Active bool
}

// SmallData uses value receiver (reasonable)
func (s SmallData) Double() SmallData {
    s.Value *= 2
    return s
}

// LargeData uses pointer receiver (reasonable)
func (l *LargeData) Activate() {
    l.Active = true
}

func main() {
    // Small data type
    small := SmallData{Value: 5}
    doubled := small.Double()
    fmt.Printf("SmallData - Original: %d, Doubled: %d\n", small.Value, doubled.Value)
    
    // Large data type
    large := LargeData{Name: "Test"}
    fmt.Printf("LargeData - Active before modification: %t\n", large.Active)
    large.Activate()
    fmt.Printf("LargeData - Active after modification: %t\n", large.Active)
}

Execution result:

SmallData - Original: 5, Doubled: 10
LargeData - Active before modification: false
LargeData - Active after modification: true

4. Receivers of Different Types

4.1 Methods for Basic Types

package main

import "fmt"

// Create alias for int type
type MyInt int

// Add methods to custom type
func (mi MyInt) Double() MyInt {
    return mi * 2
}

func (mi MyInt) IsEven() bool {
    return mi%2 == 0
}

func (mi *MyInt) Increment() {
    *mi++
}

// Create methods for built-in types (need type aliases)
type Celsius float64
type Fahrenheit float64

func (c Celsius) ToFahrenheit() Fahrenheit {
    return Fahrenheit(c*9/5 + 32)
}

func (f Fahrenheit) ToCelsius() Celsius {
    return Celsius((f - 32) * 5 / 9)
}

func main() {
    // Custom int type
    num := MyInt(10)
    fmt.Printf("Original value: %d\n", num)
    fmt.Printf("Doubled: %d\n", num.Double())
    fmt.Printf("Is even: %t\n", num.IsEven())
    
    num.Increment()
    fmt.Printf("After increment: %d\n", num)
    
    // Temperature conversion
    tempC := Celsius(25)
    tempF := tempC.ToFahrenheit()
    fmt.Printf("\n%d¬∞C = %.1f¬∞F\n", int(tempC), float64(tempF))
    
    backToC := tempF.ToCelsius()
    fmt.Printf("%.1f¬∞F = %d¬∞C\n", float64(tempF), int(backToC))
}

Execution result:

Original value: 10
Doubled: 20
Is even: true
After increment: 11

25¬∞C = 77.0¬∞F
77.0¬∞F = 25¬∞C

4.2 Struct Methods

package main

import (
    "fmt"
    "math"
)

type Point struct {
    X, Y float64
}

type Circle struct {
    Center Point
    Radius float64
}

// Point methods
func (p Point) Distance(other Point) float64 {
    dx := p.X - other.X
    dy := p.Y - other.Y
    return math.Sqrt(dx*dx + dy*dy)
}

func (p Point) Move(dx, dy float64) Point {
    return Point{X: p.X + dx, Y: p.Y + dy}
}

// Circle methods
func (c Circle) Area() float64 {
    return math.Pi * c.Radius * c.Radius
}

func (c Circle) Circumference() float64 {
    return 2 * math.Pi * c.Radius
}

func (c *Circle) Scale(factor float64) {
    c.Radius *= factor
}

func main() {
    // Point operations
    p1 := Point{X: 0, Y: 0}
    p2 := Point{X: 3, Y: 4}
    
    distance := p1.Distance(p2)
    fmt.Printf("Distance between points: %.2f\n", distance)
    
    p3 := p1.Move(1, 1)
    fmt.Printf("Point after moving: (%f, %f)\n", p3.X, p3.Y)
    
    // Circle operations
    circle := Circle{
        Center: Point{X: 0, Y: 0},
        Radius: 5,
    }
    
    fmt.Printf("\nCircle radius: %.2f\n", circle.Radius)
    fmt.Printf("Circle area: %.2f\n", circle.Area())
    fmt.Printf("Circle circumference: %.2f\n", circle.Circumference())
    
    circle.Scale(2)
    fmt.Printf("\nRadius after scaling: %.2f\n", circle.Radius)
    fmt.Printf("Area after scaling: %.2f\n", circle.Area())
}

Execution result:

Distance between points: 5.00
Point after moving: (1.000000, 1.000000)

Circle radius: 5.00
Circle area: 78.54
Circle circumference: 31.42

Radius after scaling: 10.00
Area after scaling: 314.16

4.3 Slice and Map Methods

package main

import "fmt"

// Create alias for slice type
type IntSlice []int

// Create alias for map type
type StringMap map[string]string

// IntSlice methods
func (is IntSlice) Sum() int {
    sum := 0
    for _, v := range is {
        sum += v
    }
    return sum
}

func (is IntSlice) Average() float64 {
    if len(is) == 0 {
        return 0
    }
    return float64(is.Sum()) / float64(len(is))
}

func (is *IntSlice) Append(value int) {
    *is = append(*is, value)
}

// StringMap methods
func (sm StringMap) Get(key string) (string, bool) {
    value, exists := sm[key]
    return value, exists
}

func (sm StringMap) Set(key, value string) {
    sm[key] = value
}

func (sm StringMap) Keys() []string {
    keys := make([]string, 0, len(sm))
    for k := range sm {
        keys = append(keys, k)
    }
    return keys
}

func main() {
    // Slice methods
    numbers := IntSlice{1, 2, 3, 4, 5}
    fmt.Printf("Number slice: %v\n", numbers)
    fmt.Printf("Sum: %d\n", numbers.Sum())
    fmt.Printf("Average: %.2f\n", numbers.Average())
    
    numbers.Append(6)
    numbers.Append(7)
    fmt.Printf("After adding elements: %v\n", numbers)
    fmt.Printf("New sum: %d\n", numbers.Sum())
    
    // Map methods
    studentGrades := StringMap{
        "Zhang San": "85",
        "Li Si": "92",
        "Wang Wu": "78",
    }
    
    fmt.Printf("\nStudent grades: %v\n", studentGrades)
    
    // Get value
    if grade, exists := studentGrades.Get("Li Si"); exists {
        fmt.Printf("Li Si's grade: %s\n", grade)
    }
    
    // Set new value
    studentGrades.Set("Zhao Liu", "88")
    fmt.Printf("After adding Zhao Liu: %v\n", studentGrades)
    
    // Get all keys
    keys := studentGrades.Keys()
    fmt.Printf("All students: %v\n", keys)
}

Execution result:

Number slice: [1 2 3 4 5]
Sum: 15
Average: 3.00
After adding elements: [1 2 3 4 5 6 7]
New sum: 28

Student grades: map[Zhang San:85 Li Si:92 Wang Wu:78]
Li Si's grade: 92
After adding Zhao Liu: map[Zhang San:85 Li Si:92 Wang Wu:78 Zhao Liu:88]
All students: [Zhang San Li Si Wang Wu Zhao Liu]

5. Relationship Between Methods and Interfaces

5.1 Concept of Method Sets

package main

import "fmt"

type Shape interface {
    Area() float64
    Perimeter() float64
}

type Rectangle struct {
    Width  float64
    Height float64
}

type Circle struct {
    Radius float64
}

// Rectangle implements Shape interface
func (r Rectangle) Area() float64 {
    return r.Width * r.Height
}

func (r Rectangle) Perimeter() float64 {
    return 2 * (r.Width + r.Height)
}

// Circle implements Shape interface
func (c Circle) Area() float64 {
    return 3.14159 * c.Radius * c.Radius
}

func (c Circle) Perimeter() float64 {
    return 2 * 3.14159 * c.Radius
}

// Function using interface
func PrintShapeInfo(s Shape) {
    fmt.Printf("Area: %.2f\n", s.Area())
    fmt.Printf("Perimeter: %.2f\n", s.Perimeter())
}

func main() {
    rect := Rectangle{Width: 10, Height: 5}
    circle := Circle{Radius: 3}
    
    fmt.Println("=== Rectangle ===")
    PrintShapeInfo(rect)
    
    fmt.Println("\n=== Circle ===")
    PrintShapeInfo(circle)
    
    // Interface slice
    shapes := []Shape{rect, circle}
    fmt.Println("\n=== Batch processing ===")
    for i, shape := range shapes {
        fmt.Printf("Shape %d:\n", i+1)
        PrintShapeInfo(shape)
        fmt.Println()
    }
}

Execution result:

=== Rectangle ===
Area: 50.00
Perimeter: 30.00

=== Circle ===
Area: 28.27
Perimeter: 18.85

=== Batch processing ===
Shape 1:
Area: 50.00
Perimeter: 30.00

Shape 2:
Area: 28.27
Perimeter: 18.85

6. Advanced Method Usage

6.1 Method Chaining

package main

import "fmt"

type Calculator struct {
    Result float64
}

func (c Calculator) Add(x float64) Calculator {
    c.Result += x
    return c
}

func (c Calculator) Subtract(x float64) Calculator {
    c.Result -= x
    return c
}

func (c Calculator) Multiply(x float64) Calculator {
    c.Result *= x
    return c
}

func (c Calculator) Divide(x float64) Calculator {
    if x != 0 {
        c.Result /= x
    }
    return c
}

func (c Calculator) GetValue() float64 {
    return c.Result
}

func main() {
    calc := Calculator{Result: 10}
    
    result := calc.
        Add(5).
        Multiply(2).
        Subtract(3).
        Divide(2).
        GetValue()
    
    fmt.Printf("Calculation result: %.2f\n", result)
    
    // Step-by-step demonstration
    calc2 := Calculator{Result: 10}
    fmt.Printf("Initial value: %.2f\n", calc2.GetValue())
    
    calc2 = calc2.Add(5)
    fmt.Printf("After adding 5: %.2f\n", calc2.GetValue())
    
    calc2 = calc2.Multiply(2)
    fmt.Printf("After multiplying by 2: %.2f\n", calc2.GetValue())
    
    calc2 = calc2.Subtract(3)
    fmt.Printf("After subtracting 3: %.2f\n", calc2.GetValue())
    
    calc2 = calc2.Divide(2)
    fmt.Printf("After dividing by 2: %.2f\n", calc2.GetValue())
}

Execution result:

Calculation result: 13.50
Initial value: 10.00
After adding 5: 15.00
After multiplying by 2: 30.00
After subtracting 3: 27.00
After dividing by 2: 13.50

6.2 Method Promotion in Embedded Types

package main

import "fmt"

type Person struct {
    Name string
    Age  int
}

func (p Person) GetName() string {
    return p.Name
}

func (p Person) GetAge() int {
    return p.Age
}

type Student struct {
    Person      // Embed Person
    School string
    Grade  int
}

func (s Student) GetSchool() string {
    return s.School
}

type Teacher struct {
    Person      // Embed Person
    Subject string
    Salary  float64
}

func (t Teacher) GetSubject() string {
    return t.Subject
}

func main() {
    student := Student{
        Person: Person{Name: "Xiao Ming", Age: 18},
        School: "No.1 Middle School",
        Grade:  12,
    }
    
    teacher := Teacher{
        Person:  Person{Name: "Teacher Wang", Age: 35},
        Subject: "Mathematics",
        Salary:  8000,
    }
    
    // Can directly call embedded type methods
    fmt.Printf("Student name: %s\n", student.GetName())    // Promoted method
    fmt.Printf("Student age: %d\n", student.GetAge())     // Promoted method
    fmt.Printf("School: %s\n", student.GetSchool())
    fmt.Printf("Grade: %d\n", student.Grade)
    
    fmt.Printf("\nTeacher name: %s\n", teacher.GetName())  // Promoted method
    fmt.Printf("Teacher age: %d\n", teacher.GetAge())     // Promoted method
    fmt.Printf("Subject: %s\n", teacher.GetSubject())
    fmt.Printf("Salary: %.2f\n", teacher.Salary)
    
    // Verify type relationship
    fmt.Printf("\nStudent is also Person type: %t\n", isPerson(student))
    fmt.Printf("Teacher is also Person type: %t\n", isPerson(teacher))
}

func isPerson(p Person) bool {
    return true
}

Execution result:

Student name: Xiao Ming
Student age: 18
School: No.1 Middle School
Grade: 12

Teacher name: Teacher Wang
Teacher age: 35
Subject: Mathematics
Salary: 8000.00

Student is also Person type: true
Teacher is also Person type: true

7. Method Considerations

7.1 Method Name Conflicts

package main

import "fmt"

type Base struct {
    Name string
}

func (b Base) GetName() string {
    return b.Name
}

type Derived struct {
    Base
    Name string  // Same name as Name in Base
}

func (d Derived) GetName() string {
    // Explicitly specify which GetName to call
    return "Derived: " + d.Name + ", Base: " + d.Base.GetName()
}

func main() {
    derived := Derived{
        Base: Base{Name: "BaseName"},
        Name: "DerivedName",
    }
    
    fmt.Printf("Calling derived class GetName: %s\n", derived.GetName())
    fmt.Printf("Direct access to Name: %s\n", derived.Name)
    fmt.Printf("Access Base.Name: %s\n", derived.Base.Name)
}

Execution result:

Calling derived class GetName: Derived: DerivedName, Base: BaseName
Direct access to Name: DerivedName
Access Base.Name: BaseName

7.2 Calling Methods on Nil Pointers

package main

import "fmt"

type Data struct {
    Value int
}

func (d *Data) GetValue() int {
    if d == nil {
        return 0  // Safety check
    }
    return d.Value
}

func (d *Data) SetValue(v int) {
    if d != nil {  // Safety check
        d.Value = v
    }
}

func main() {
    var ptr *Data = nil
    
    // Calling method on nil pointer (safe)
    fmt.Printf("Nil pointer GetValue: %d\n", ptr.GetValue())
    
    // Calling modification method on nil pointer (safe)
    ptr.SetValue(100)  // No problem because of safety check
    
    // Create actual object
    actual := &Data{Value: 42}
    fmt.Printf("Actual object GetValue: %d\n", actual.GetValue())
    
    actual.SetValue(99)
    fmt.Printf("GetValue after modification: %d\n", actual.GetValue())
}

Execution result:

Nil pointer GetValue: 0
Actual object GetValue: 42
GetValue after modification: 99

8. Summary

Methods are the core mechanism for implementing object-oriented programming in Go language:

Core concepts:

  • Methods are functions bound to specific types
  • Associate methods with types through receivers
  • Value receivers and pointer receivers have different behavioral characteristics

Usage principles:

  • Value receivers: Suitable for not modifying data, small data types
  • Pointer receivers: Suitable for needing to modify data, large data types
  • Maintain consistency in receiver types for all methods of a type

Important features:

  • Method promotion: Embedded type methods can be called directly
  • Interface implementation: Methods are the foundation for implementing interfaces
  • Chaining: Returning receiver type supports method chaining

Mastering method usage is key to understanding Go language’s object-oriented features and lays a solid foundation for subsequent interface learning.
