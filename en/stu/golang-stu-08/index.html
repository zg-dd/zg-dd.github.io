1. What is a Pointer

A pointer is an important concept in programming. It acts like an “address label” that stores the specific location information of a variable in memory. Through pointers, we can directly access and manipulate data in memory.

1.1 Basic Concepts of Pointers

Imagine a real-life scenario:

  • Your home has a specific address (for example: xxx Street, Chaoyang District, Beijing)
  • Others can find your home just by knowing this address
  • A pointer is equivalent to this “address” - it stores the memory location of a variable

In computers:

  • Each variable is allocated a piece of memory space to store data
  • This memory space has a unique address
  • Pointer variables store this memory address

1.2 Why Do We Need Pointers

Pointers have several important roles:

  1. Memory Saving: When passing large data structures, passing pointers is more efficient than passing the entire data
  2. Direct Modification: Original data can be modified directly within functions
  3. Dynamic Memory Management: Memory can be allocated and freed dynamically
  4. Data Structure Implementation: Foundation for implementing complex data structures like linked lists and trees

2. Pointer Declaration and Usage

2.1 Pointer Declaration

// Pointer declaration syntax: var pointer_variable_name *data_type
var ptr *int     // Declare a pointer to int type
var ptr2 *string // Declare a pointer to string type
var ptr3 *float64 // Declare a pointer to float64 type

2.2 Getting Variable Addresses (& Operator)

package main

import "fmt"

func main() {
    // Declare ordinary variables
    var num int = 100
    var name string = "Zhang San"
    var score float64 = 95.5
    
    // Use & operator to get variable's memory address
    var ptrNum *int = &num
    var ptrName *string = &name
    var ptrScore *float64 = &score
    
    fmt.Printf("Variable num value: %d, address: %p\n", num, &num)
    fmt.Printf("Variable name value: %s, address: %p\n", name, &name)
    fmt.Printf("Variable score value: %f, address: %p\n", score, &score)
    
    fmt.Printf("Pointer ptrNum stores address: %p, pointed value: %d\n", ptrNum, *ptrNum)
    fmt.Printf("Pointer ptrName stores address: %p, pointed value: %s\n", ptrName, *ptrName)
    fmt.Printf("Pointer ptrScore stores address: %p, pointed value: %f\n", ptrScore, *ptrScore)
}

Execution result:

Variable num value: 100, address: 0xc0000140a8
Variable name value: Zhang San, address: 0xc000010230
Variable score value: 95.500000, address: 0xc0000140b8
Pointer ptrNum stores address: 0xc0000140a8, pointed value: 100
Pointer ptrName stores address: 0xc000010230, pointed value: Zhang San
Pointer ptrScore stores address: 0xc0000140b8, pointed value: 95.500000

2.3 Accessing Values Pointed by Pointers (* Operator)

package main

import "fmt"

func main() {
    var num int = 50
    var ptr *int = &num  // ptr points to num's address
    
    fmt.Printf("num value: %d\n", num)
    fmt.Printf("ptr stores address: %p\n", ptr)
    fmt.Printf("Value accessed through pointer: %d\n", *ptr)  // *ptr represents getting the value at the address ptr points to
    
    // Modify value through pointer
    *ptr = 100
    fmt.Printf("num value after modification: %d\n", num)  // num's value is also modified
    fmt.Printf("Modified value accessed through pointer: %d\n", *ptr)
}

Execution result:

num value: 50
ptr stores address: 0xc0000140a8
Value accessed through pointer: 50
num value after modification: 100
Modified value accessed through pointer: 100

3. Pointer Zero Values and nil

3.1 Pointer Zero Values

package main

import "fmt"

func main() {
    // Declare pointer variables without initialization
    var ptr *int
    var ptr2 *string
    
    fmt.Printf("Uninitialized int pointer: %v, is nil: %t\n", ptr, ptr == nil)
    fmt.Printf("Uninitialized string pointer: %v, is nil: %t\n", ptr2, ptr2 == nil)
    
    // Trying to access values of uninitialized pointers will cause panic

3.2 Safe Checking of nil Pointers

package main

import "fmt"

func main() {
    var ptr *int
    
    // Safety check
    if ptr != nil {
        fmt.Println("Pointer is not nil, safe to access")
        fmt.Println(*ptr)
    } else {
        fmt.Println("Pointer is nil, cannot access")
    }
    
    // After initializing pointer
    num := 42
    ptr = &num
    
    if ptr != nil {
        fmt.Println("Pointer is not nil, safe to access")
        fmt.Println("Value pointed by pointer: ", *ptr)
    }
}

Execution result:

Pointer is nil, cannot access
Pointer is not nil, safe to access
Value pointed by pointer:  42

4. Pointers and Functions

4.1 Pointers as Function Parameters

package main

import "fmt"

// Pass by value - won't modify original value
func changeValueByValue(x int) {
    x = 100
    fmt.Printf("Value of x inside function: %d\n", x)
}

// Pass by pointer - can modify original value
func changeValueByPointer(ptr *int) {
    *ptr = 200
    fmt.Printf("Value modified through pointer inside function: %d\n", *ptr)
}

func main() {
    num := 50
    fmt.Printf("num value before calling function: %d\n", num)
    
    // Call by value
    changeValueByValue(num)
    fmt.Printf("num value after call by value: %d\n", num)
    
    // Call by pointer
    changeValueByPointer(&num)
    fmt.Printf("num value after call by pointer: %d\n", num)
}

Execution result:

num value before calling function: 50
Value of x inside function: 100
num value after call by value: 50
Value modified through pointer inside function: 200
num value after call by pointer: 200

4.2 Functions Returning Pointers

package main

import "fmt"

// Return pointer to int
func createIntPointer(value int) *int {
    return &value
}

// Return pointer to struct
func createStudentPointer(name string, age int) *struct {
    Name string
    Age  int
} {
    student := struct {
        Name string
        Age  int
    }{name, age}
    return &student
}

func main() {
    // Get pointer
    ptr := createIntPointer(999)
    fmt.Printf("Pointer address: %p, pointed value: %d\n", ptr, *ptr)
    
    // Modify value accessed through pointer
    *ptr = 888
    fmt.Printf("Modified value: %d\n", *ptr)
    
    // Struct pointer
    studentPtr := createStudentPointer("Li Si", 20)
    fmt.Printf("Student info: %s, %d years old\n", (*studentPtr).Name, (*studentPtr).Age)
    
    // Go provides simplified syntax
    fmt.Printf("Student info simplified syntax: %s, %d years old\n", studentPtr.Name, studentPtr.Age)
}

Execution result:

Pointer address: 0xc0000140a8, pointed value: 999
Modified value: 888
Student info: Li Si, 20 years old
Student info simplified syntax: Li Si, 20 years old

5. Pointer Arrays and Array Pointers

5.1 Pointer Arrays

package main

import "fmt"

func main() {
    // Create several variables
    a, b, c := 1, 2, 3
    
    // Pointer array - each element of the array is a pointer
    ptrArray := [3]*int{&a, &b, &c}
    
    fmt.Println("Pointer array content:")
    for i, ptr := range ptrArray {
        fmt.Printf("Index %d: address %p, value %d\n", i, ptr, *ptr)
    }
    
    // Modify original variables through pointer array
    *ptrArray[0] = 10
    *ptrArray[1] = 20
    *ptrArray[2] = 30
    
    fmt.Printf("After modification: a=%d, b=%d, c=%d\n", a, b, c)
}

Execution result:

Pointer array content:
Index 0: address 0xc0000140a8, value 1
Index 1: address 0xc0000140b0, value 2
Index 2: address 0xc0000140b8, value 3
After modification: a=10, b=20, c=30

5.2 Array Pointers

package main

import "fmt"

func main() {
    // Ordinary array
    arr := [3]int{10, 20, 30}
    
    // Array pointer - pointer pointing to the entire array
    arrPtr := &arr
    
    fmt.Printf("Array address: %p\n", &arr)
    fmt.Printf("Array pointer stores address: %p\n", arrPtr)
    fmt.Printf("Access elements through array pointer: %d, %d, %d\n", 
               (*arrPtr)[0], (*arrPtr)[1], (*arrPtr)[2])
    
    // Modify array elements
    (*arrPtr)[0] = 100
    (*arrPtr)[1] = 200
    (*arrPtr)[2] = 300
    
    fmt.Printf("Array after modification: %v\n", arr)
    
    // Go's simplified syntax
    arrPtr[0] = 1000  // Equivalent to (*arrPtr)[0] = 1000
    fmt.Printf("After modification with simplified syntax: %v\n", arr)
}

Execution result:

Array address: 0xc000016060
Array pointer stores address: 0xc000016060
Access elements through array pointer: 10, 20, 30
Array after modification: [100 200 300]
After modification with simplified syntax: [1000 200 300]

6. Pointers to Pointers (Multi-level Pointers)

package main

import "fmt"

func main() {
    num := 42
    
    // First level pointer
    ptr1 := &num
    fmt.Printf("num value: %d, address: %p\n", num, &num)
    fmt.Printf("First level pointer ptr1: %p, pointed value: %d\n", ptr1, *ptr1)
    
    // Second level pointer (pointer to pointer)
    ptr2 := &ptr1
    fmt.Printf("Second level pointer ptr2: %p, pointed address: %p, final value: %d\n", 
               ptr2, *ptr2, **ptr2)
    
    // Third level pointer
    ptr3 := &ptr2
    fmt.Printf("Third level pointer ptr3: %p, final value: %d\n", ptr3, ***ptr3)
    
    // Modify value through multi-level pointer
    ***ptr3 = 999
    fmt.Printf("num value after modification through third level pointer: %d\n", num)
}

Execution result:

num value: 42, address: 0xc0000140a8
First level pointer ptr1: 0xc0000140a8, pointed value: 42
Second level pointer ptr2: 0xc000006028, pointed address: 0xc0000140a8, final value: 42
Third level pointer ptr3: 0xc000006038, final value: 42
num value after modification through third level pointer: 999

7. Practical Application Examples of Pointers

7.1 Swapping Two Numbers

package main

import "fmt"

// Swap two numbers using pointers
func swap(a, b *int) {
    temp := *a
    *a = *b
    *b = temp
}

func main() {
    x, y := 10, 20
    fmt.Printf("Before swap: x=%d, y=%d\n", x, y)
    
    swap(&x, &y)
    fmt.Printf("After swap: x=%d, y=%d\n", x, y)
}

Execution result:

Before swap: x=10, y=20
After swap: x=20, y=10

7.2 Linked List Node Example

package main

import "fmt"

// Define linked list node structure
type ListNode struct {
    Value int
    Next  *ListNode
}

// Create new node
func NewNode(value int) *ListNode {
    return &ListNode{Value: value, Next: nil}
}

// Add node at the end of linked list
func AppendNode(head **ListNode, value int) {
    newNode := NewNode(value)
    
    if *head == nil {
        *head = newNode
        return
    }
    
    current := *head
    for current.Next != nil {
        current = current.Next
    }
    current.Next = newNode
}

// Print linked list
func PrintList(head *ListNode) {
    current := head
    for current != nil {
        fmt.Printf("%d -> ", current.Value)
        current = current.Next
    }
    fmt.Println("nil")
}

func main() {
    var head *ListNode = nil
    
    // Add nodes
    AppendNode(&head, 1)
    AppendNode(&head, 2)
    AppendNode(&head, 3)
    AppendNode(&head, 4)
    
    fmt.Println("Linked list content:")
    PrintList(head)
}

Execution result:

Linked list content:
1 -> 2 -> 3 -> 4 -> nil

8. Pointer Considerations and Best Practices

8.1 Important Notes

  1. Danger of nil pointer access: Accessing nil pointers will cause program crashes
  2. Dangling pointer problem: Pointers pointing to released memory
  3. Memory leak: Forgetting to free dynamically allocated memory

8.2 Best Practices

package main

import "fmt"

func main() {
    // 1. Always check if pointer is nil
    var ptr *int
    if ptr != nil {
        fmt.Println(*ptr) // Safe access
    }
    
    // 2. Initialize pointers
    num := 100
    ptr = &num
    fmt.Printf("Safe access: %d\n", *ptr)
    
    // 3. Consider lifetime when using pointers
    data := make([]int, 5)
    ptrToSlice := &data
    fmt.Printf("Slice pointer: %v\n", *ptrToSlice)
    
    // 4. Be careful when functions return pointers
    func returnPointer() *int {
        value := 42  // Local variable
        return &value // Dangerous! Returning address of local variable
    }
    
    // Correct approach
    func returnPointerSafe() *int {
        value := new(int) // Allocate memory on heap
        *value = 42
        return value
    }
}

9. Difference Between new and make Functions

9.1 new Function

package main

import "fmt"

func main() {
    // new function allocates zero-value memory for specified type and returns pointer
    ptr1 := new(int)        // *int type, value 0
    ptr2 := new(string)     // *string type, value ""
    ptr3 := new([3]int)     // *[3]int type, value [0,0,0]
    
    fmt.Printf("int pointer: %p, value: %d\n", ptr1, *ptr1)
    fmt.Printf("string pointer: %p, value: %s\n", ptr2, *ptr2)
    fmt.Printf("array pointer: %p, value: %v\n", ptr3, *ptr3)
    
    // Modify values
    *ptr1 = 100
    *ptr2 = "Hello"
    (*ptr3)[0] = 1
    (*ptr3)[1] = 2
    (*ptr3)[2] = 3
    
    fmt.Printf("Modified int value: %d\n", *ptr1)
    fmt.Printf("Modified string value: %s\n", *ptr2)
    fmt.Printf("Modified array value: %v\n", *ptr3)
}

Execution result:

int pointer: 0xc0000140a8, value: 0
string pointer: 0xc000010230, value: 
array pointer: 0xc000016060, value: [0 0 0]
Modified int value: 100
Modified string value: Hello
Modified array value: [1 2 3]

9.2 make Function

package main

import "fmt"

func main() {
    // make function is used to create slices, maps, channels
    slice := make([]int, 5)        // Create slice with length 5
    mapData := make(map[string]int) // Create map
    ch := make(chan int)           // Create channel
    
    fmt.Printf("Slice: %v, length: %d\n", slice, len(slice))
    fmt.Printf("Map: %v\n", mapData)
    fmt.Printf("Channel: %v\n", ch)
    
    // Initialize data
    slice[0] = 10
    slice[1] = 20
    mapData["one"] = 1
    mapData["two"] = 2
    
    fmt.Printf("Slice after initialization: %v\n", slice)
    fmt.Printf("Map after initialization: %v\n", mapData)
}

10. Summary

Pointers are a powerful but cautious feature in Golang:

Core Concepts:

  • Pointers store memory addresses, not actual values
  • Use & to get variable address, use * to access value pointed by pointer
  • The zero value of pointers is nil

Main Uses:

  1. Sharing and modifying data between functions
  2. Avoiding copy overhead of large data structures
  3. Implementing complex data structures (linked lists, trees, etc.)
  4. Dynamic memory management

Important Notes:

  • Always check if pointers are nil
  • Pay attention to variable lifecycles
  • Avoid dangling pointers and memory leaks

Mastering pointers is very important for deeply understanding Golang and writing efficient programs. Although it may seem complex when first learning, it can be mastered through extensive practice.
