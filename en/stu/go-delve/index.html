Go Debugger Delve (dlv) Notes

Overview

Delve (dlv) is the official Go debugger, designed for Go‚Äôs concurrency model and runtime. It is the most widely used and powerful debugger in the Go ecosystem.

Installation

Recommended install

Install the latest version using Go modules:

go install github.com/go-delve/delve/cmd/dlv@latest

Verify

After installation:

dlv version

If $GOPATH/bin is not in your PATH, add it or use the full path:

~/go/bin/dlv version

Core capabilities

  • Breakpoints at specific lines or functions
  • Step execution (line-by-line or into functions)
  • Variable inspection and complex structure viewing
  • Goroutine management (inspect and switch)
  • Runtime variable changes for testing
  • Call stack inspection

Basic usage

Example code

Create main.go:

package main

import "fmt"

func main() {
   message := "Hello, Delve!"
   fmt.Println(message)
   result := add(23, 19)
   fmt.Printf("Result: %d\n", result)
}

func add(a, b int) int {
   sum := a + b
   return sum
}

Start a debug session

Mode 1: Debug source

dlv debug main.go

Mode 2: Debug binary

go build -o myapp main.go
dlv exec ./myapp

Mode 3: Attach to running process

# Build and run
go build -gcflags="all=-N -l" -o myapp main.go
./myapp &

# Attach
PID=$!
dlv attach $PID

Advanced features

Conditional breakpoints

(dlv) b main.go:10
Breakpoint 1 set at main.go:10
(dlv) cond 1 i == 5

Debug tests

dlv test main_test.go

Set breakpoint:

(dlv) b TestMyFunction

Pass command-line args

dlv debug -- arg1 arg2
# or
dlv exec ./myapp -- arg1 arg2

Goroutine debugging

(dlv) goroutines
(dlv) goroutine 5
(dlv) p localVar

Practical tips

1. Production debugging

Build with optimizations disabled:

go build -gcflags="all=-N -l" -o myapp main.go

2. Performance considerations

  • Debug builds run slower; use them only for debugging.
  • Use standard build options in production.

3. IDE integration

  • VS Code: install Go extension
  • Goland: built-in Delve support
  • CLI proficiency helps you understand IDE debugging

4. Troubleshooting

Issue: breakpoints don‚Äôt hit

  • Ensure -gcflags="all=-N -l"
  • Verify breakpoint line and function

Issue: variable values look wrong

  • Compiler optimizations can distort values
  • Rebuild with optimizations disabled

Summary

Delve is essential for Go developers. With it you can:

  1. Locate bugs quickly using breakpoints and stepping
  2. Understand code flow by tracking variables
  3. Debug concurrency by inspecting goroutines
  4. Improve productivity by reducing debug time

Start with small programs, practice the commands, and gradually apply Delve to complex debugging tasks.
