Chapter 1: What are Skills

1.1 Introduction to Skills

Skills are an extension system for Claude Code, similar to installing “skill packs” for Claude.

Simple Understanding:

  • Like mobile apps: you can install different apps on your phone to achieve different functions
  • Skills are “functional plugins” installed for Claude
  • Each Skill teaches Claude a new capability

Examples:

  • Install docx skill ‚Üí Claude can create and edit Word documents
  • Install pdf skill ‚Üí Claude can process PDF files
  • Install mysql-manager skill ‚Üí Claude can operate databases

1.2 Types of Skills

Official Skills (Provided by Anthropic)

  • Developed and maintained by Claude’s official team
  • Guaranteed quality and stable functionality
  • Free to use
  • Examples: docx, pdf, pptx, xlsx, etc.

Community Skills (Third-party Development)

  • Contributed by community developers
  • Diverse functionalities
  • Quality needs to be evaluated independently

Custom Skills (Created by You)

  • Customized according to your needs
  • Fully controllable
  • This tutorial will teach you how to create them

1.3 Why Use Skills

Without Skills:

You: Help me create a Word document
Claude: Sorry, I cannot directly create Word documents

With Skills:

You: Help me create a Word document
Claude: OK! (Automatically uses docx skill to create document)

Advantages of Skills:

  1. ‚úÖ Extend Claude’s capabilities
  2. ‚úÖ Automate repetitive tasks
  3. ‚úÖ Improve work efficiency
  4. ‚úÖ Can customize exclusive functions

1.4 How Skills Work

Your request ‚Üí Claude analyzes ‚Üí Select appropriate Skill ‚Üí Execute operation ‚Üí Return results

Example Process:

  1. You say: “Create a Word document”
  2. Claude identifies that document processing is needed
  3. Claude calls the docx skill
  4. docx skill executes document creation code
  5. Returns the created document

Chapter 4: Using Skills

4.1 Three Ways to Use Skills

Method 1: Natural Language (Recommended for Beginners)

Directly describe your needs in everyday language, Claude will automatically select the appropriate skill.

Examples:

Create a Word document
Process this PDF file
Design a login page
Query database

Advantages:

  • Simplest, no need to remember skill names
  • Claude automatically determines which skill to use
  • Suitable for daily use

Disadvantages:

  • Sometimes Claude might choose the wrong skill
  • Not precise enough

Method 2: Explicitly Specify Skill

If you know which skill to use, you can specify it explicitly.

Examples:

Use docx skill to create report
Use pdf skill to extract text
Use mysql-manager to query data

Advantages:

  • More precise
  • Avoids Claude choosing wrong skill
  • Suitable for repetitive tasks

Disadvantages:

  • Need to remember skill names

Method 3: Direct Command Call (Advanced Users)

For custom skills, can call commands directly.

Examples:

cd ~/.claude/skills/mysql-manager
node mysql-tool.js list-databases

Advantages:

  • Fastest
  • Full control
  • Suitable for scripting

Disadvantages:

  • Need command-line knowledge
  • Need to know specific commands

6.4 Creating Core Tool Script

mysql-tool.js (Core functionality implementation):

#!/usr/bin/env node

const mysql = require('mysql2/promise');
const fs = require('fs').promises;
const path = require('path');

// Load configuration
async function loadConfig() {
  const configPath = path.join(__dirname, 'config.json');
  const configData = await fs.readFile(configPath, 'utf8');
  return JSON.parse(configData);
}

// Create database connection
async function createConnection(dbName = null) {
  const config = await loadConfig();
  if (dbName) {
    config.database = dbName;
  }
  return await mysql.createConnection(config);
}

// List all databases
async function listDatabases() {
  const conn = await createConnection();
  const [rows] = await conn.execute('SHOW DATABASES');
  await conn.end();
  return rows.map(row => row.Database);
}

// List all tables
async function listTables(dbName) {
  const conn = await createConnection(dbName);
  const [rows] = await conn.execute('SHOW TABLES');
  await conn.end();
  return rows.map(row => Object.values(row)[0]);
}

// Query data
async function selectAll(dbName, tableName, limit = 100) {
  const conn = await createConnection(dbName);
  const [rows] = await conn.execute(
    `SELECT * FROM \`${tableName}\` LIMIT ?`,
    [limit]
  );
  await conn.end();
  return rows;
}

// Main function
async function main() {
  const args = process.argv.slice(2);
  const command = args[0];

  try {
    let result;

    switch (command) {
      case 'list-databases':
        result = await listDatabases();
        break;
      case 'list-tables':
        result = await listTables(args[1]);
        break;
      case 'select-all':
        result = await selectAll(args[1], args[2], parseInt(args[3] || 100));
        break;
      default:
        console.error('Unknown command:', command);
        process.exit(1);
    }

    console.log(JSON.stringify(result, null, 2));
  } catch (error) {
    console.error('Error:', error.message);
    process.exit(1);
  }
}

if (require.main === module) {
  main();
}

Set Execution Permissions:

chmod +x mysql-tool.js

6.8 Extending Functions

Adding More Commands:

  1. Create Database:
async function createDatabase(dbName) {
  const conn = await createConnection();
  await conn.execute(`CREATE DATABASE IF NOT EXISTS \`${dbName}\``);
  await conn.end();
  return `Database ${dbName} created successfully`;
}
  1. Export to CSV:
async function exportToCSV(dbName, tableName, outputPath) {
  const conn = await createConnection(dbName);
  const [rows] = await conn.execute(`SELECT * FROM \`${tableName}\``);
  await conn.end();

  const headers = Object.keys(rows[0]).join(',');
  const csvRows = rows.map(row =>
    Object.values(row).map(val =>
      typeof val === 'string' ? `"${val}"` : val
    ).join(',')
  );

  const csv = [headers, ...csvRows].join('\n');
  await fs.writeFile(outputPath, csv, 'utf8');

  return `Export successful: ${outputPath}`;
}
  1. Backup Database:
async function backupDatabase(dbName, outputPath) {
  const cmd = `mysqldump -u ${config.user} -p${config.password} ${dbName} > "${outputPath}"`;
  await execPromise(cmd);
  return `Backup successful: ${outputPath}`;
}

Chapter 7: Common Problems and Solutions

7.1 Installation Issues

Problem 1: Cannot Add Official Marketplace

Error Message:

Failed to add marketplace

Possible Causes:

  • Network connection issues
  • Cannot access GitHub
  • Proxy configuration issues

Solutions:

  1. Check network connection:
ping github.com
  1. Configure Git proxy (if in China):
git config --global http.proxy http://proxy.example.com:8080
git config --global https.proxy https://proxy.example.com:8080
  1. Use VPN or proxy

Problem 2: Skills Don’t Show After Installation

Symptoms:

  • Installation successful but Claude doesn’t see skills
  • Asking “What skills do you have?” shows nothing

Solutions:

  1. Restart Claude Code (most common cause):
# Exit
exit

# Restart
claude-code
  1. Check Installation Location:
ls ~/.claude/plugins/cache/
  1. Reinstall:
/plugin uninstall example-skills
/plugin install example-skills@anthropic-agent-skills

Problem 3: Dependency Installation Failed

Error Message:

npm ERR! Cannot find module 'mysql2'

Solutions:

  1. Enter skill directory:
cd ~/.claude/skills/mysql-manager
  1. Install dependencies:
npm install mysql2
  1. If still failing, clear cache:
npm cache clean --force
npm install

7.4 Performance Issues

Problem 9: Queries Are Slow

Symptoms:

  • Querying large tables responds slowly
  • Claude takes long time to respond

Solutions:

  1. Limit query count:
// Default limit 100 rows
async function selectAll(dbName, tableName, limit = 100) {
  // ...
}
  1. Add indexes:
CREATE INDEX idx_name ON users(name);
  1. Use pagination:
async function selectPage(dbName, tableName, page, pageSize) {
  const offset = (page - 1) * pageSize;
  const [rows] = await conn.execute(
    `SELECT * FROM \`${tableName}\` LIMIT ? OFFSET ?`,
    [pageSize, offset]
  );
  return rows;
}

Problem 10: High Memory Usage

Symptoms:

  • Node.js process occupies large memory
  • System becomes slow

Solutions:

  1. Use streaming processing:
const stream = conn.query(`SELECT * FROM large_table`).stream();
stream.on('data', (row) => {
  // Process each row
});
  1. Close connections promptly:
try {
  // Operations
} finally {
  await conn.end();
}
  1. Limit connection pool size:
{
  "connectionLimit": 5
}
