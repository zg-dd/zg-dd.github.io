Preface: Packages in Go are collections of functionality provided either by the standard library or by custom code. You need to import a package before using itâ€”for today we focus on the fmt package, imported with import "fmt".

1. fmt Package

The fmt package is a widely used standard library module that handles formatted I/O, similar to printf and scanf in C. It lets you read input and display formatted data to stdout or to other writers.

1.1 Printing

Printing in Go is handled by functions whose names start with Print. Print writes without inserting a newline, Println appends a newline, and Printf allows formatted printing.

1.1.1 fmt.Print 

// package declaration: think of it as a folder
package main

// import statement: bring in the fmt package
import "fmt"

// main function: entry point
func main() {
    fmt.Print('C')           // prints a rune; single quotes are flexible
    fmt.Print("a")         // prints a string
    fmt.Print("Hello World") // prints a string in double quotes
    fmt.Print(123)           // prints an integer
    fmt.Print(3.1415926)     // prints a float
    fmt.Print(true)          // prints a boolean
}

Output:

67aHello World1233.1415926true

Summary: Print can print anything. Note that 'C' prints the ASCII value 67 because single quotes denote a rune literal; double quotes produce a string. Writing 'ABC' would trigger the error more than one character in rune literal because rune literals must be a single character.

2. Variables

Variables are containers for data and can hold any type of value. Variable names must consist of letters, digits, and underscores, and cannot start with a digit. Keywords and reserved words cannot be used as variable names.

2.1 Variable declaration

Declare a variable by defining it. Several options exist:

2.3 Anonymous variables

Anonymous variables discard values using _ and do not occupy namespace.

Example:

package main

import "fmt"

func getUserInfo() (string, int) {
    return "zhangsan", 10
}

func main() {
    var name, age = getUserInfo()
    fmt.Println(name, age)
    var _, age1 = getUserInfo()
    fmt.Println(age1)
    var name1, _ = getUserInfo()
    fmt.Println(name1)
}

Result:

zhangsan 10
10
zhangsan

Anonymous variables can be reused because they do not reserve names, but using var or := requires unique names.

3. Constants

Variables change over time; constants never do. Use const to declare them.

3.1 Declaring constants 

const A = "TEST1"
fmt.Println(A)

const (
    B = "TEST2"
    C = "TEST3"
    D = "TEST4"
)
fmt.Println(B, C, D)

const (
    E = "TEST5"
    F
    G
    H = "TEST6"
    I
)
fmt.Println(E, F, G, H, I)

Output:

TEST1
TEST2 TEST3 TEST4
TEST5 TEST5 TEST5 TEST6 TEST6

3.2 iota counter

iota is a counter used together with const.

const (
    n1          = iota
    n2
    _
    n3
    n4          = iota
    n5
    n6          = 100
    n7          = iota
    n8, n9, n10 = iota, iota, iota + 1
)
fmt.Println(n1, n2, n3, n4, n5, n6, n7, n8, n9, n10)

Output:

0 1 3 4 5 100 7 8 8 9

4. Summary: Variable and constant naming rules

  1. Variable names may contain letters, digits, and underscores.
  2. Identifiers cannot begin with a digit.
  3. Identifiers cannot be reserved keywords (var, if, for, etc.).
  4. Names are case-sensitive: name, Name, NAme, and NAME are different.
  5. Choose descriptive names.
  6. CamelCase is preferred; for example maxAge. Acronyms such as DNS or IP can stay uppercase.
