2. Process/Thread model

Process approach:

  • Use waitpid() to avoid zombie processes
  • Parent must reap child resources

Zombie handling details:

  1. Cause:

    • Child terminates but parent never calls wait()
    • Consumes process table resources

  2. Solutions:

    • Synchronous reap:
      waitpid(pid, NULL, 0);
    • Async reap (recommended):
      signal(SIGCHLD, [](int) {
    while(waitpid(-1, NULL, WNOHANG) > 0);
});

  3. Special cases:

    • If parent exits first, init adopts and reaps
    • Batch reaping requires a loop

Thread approach:

  • Must synchronize shared variables
  • More complex critical‚Äësection control

Core mechanism:

  1. Registration via epoll_ctl (red‚Äëblack tree)
  2. Ready list maintained by kernel

Principle:

  • No user‚Äëthread polling

Typical use cases:
